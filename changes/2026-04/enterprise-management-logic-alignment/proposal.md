# Enterprise Management Logic Alignment Proposal

## 1. 目标

修正企业管理模块的 canonical openspec 口径，使其与 [`docs/04-modules/01-系统管理模块/模块PRD.md`](/E:/demo0413/docs/04-modules/01-系统管理模块/模块PRD.md) 和 [`docs/04-modules/01-系统管理模块/数据库设计.md`](/E:/demo0413/docs/04-modules/01-系统管理模块/数据库设计.md) 保持一致。

本次 change 明确企业管理不是“后端缺口占位页”，而是一个真实存在的“企业 + 企业管理员”一体化管理能力；同时要求界面保持不变，只修正业务逻辑、接口契约、权限和租户隔离语义。

## 2. 本次做什么

- 修正现有 enterprise 模块的错误规格口径：
  - 不再把企业管理定义为“仅占位”
  - 改为冻结真实列表、表单、保存、删除、权限、隔离规则
- 冻结企业管理页面能力：
  - 关键字搜索（企业名称 / 管理员 / 登录账号）
  - 企业类型筛选
  - 状态筛选
  - 分页列表
  - 新增企业
  - 编辑企业
  - 删除企业
- 冻结企业保存逻辑：
  - 新增企业时同步创建企业管理员
  - 企业编码由后端生成，不允许手工编辑
  - 编辑时密码留空表示不重置
- 冻结删除守卫：
  - 若企业下仍存在有效的非管理员人员账号，则禁止删除
- 冻结权限与租户隔离：
  - `enterprise_add` / `tenant_add`
  - `enterprise_edit` / `tenant_edit`
  - `enterprise_delete` / `tenant_delete`
  - 新增 / 删除仅平台超管可见
  - 后端作为越权与租户范围的最终裁判
- 把企业管理与人员管理、角色管理的联动关系写入 spec：
  - 企业管理员关系
  - 企业类型驱动管理员角色解析
  - 删除守卫依赖人员有效状态

## 3. 本次不做什么

- 不改企业管理页现有界面结构、布局、字段顺序、列表列顺序
- 不新增新的页面、分步向导或二级详情页
- 不扩展未在模块文档中出现的新字段、新状态、新流程
- 不改人员管理、角色管理的界面，仅冻结和企业管理强耦合的逻辑联动

## 5. 风险

- 当前仓库检索未直接定位到企业管理后端 Java 实现锚点，本 change 以模块文档为规格真源；代码落地时必须再次校准 Controller / Service / Mapper / SQL。
- “界面保持不变”会限制交互层面的修复方式，因此本次只能修正语义与行为，不能通过改布局规避逻辑问题。
- 删除守卫、企业管理员关系、企业类型角色映射属于跨模块联动逻辑，若只改单点，极易留下闭环缺口。

## 6. 验证

- 企业列表查询、分页、筛选
- 新增企业并同步创建企业管理员
- 编辑企业且密码留空不重置
- 企业编码后端生成且不可手工修改
- 删除守卫阻止误删
- 平台超管与非平台超管权限差异
- 租户查询越权与写越权拦截

## 7. 回滚

- 若后续实现与真实后端代码冲突，可先回滚本 change 对 enterprise 口径的新增 spec，并恢复到上一版企业占位口径。
- 但回滚前必须记录冲突点，不能直接恢复为“企业管理不存在”这一明显错误的规格假设。
