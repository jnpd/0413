# Enterprise Management Logic Alignment Test Cases

## 1. 列表与查询

### TC-ENT-001 关键字搜索命中企业名称

- 前置：存在多个企业数据
- 步骤：
  1. 输入企业名称关键字
  2. 触发搜索
- 期望：
  - 调用企业分页接口
  - 返回结果只包含匹配企业名称的数据

### TC-ENT-002 关键字搜索命中管理员姓名或登录账号

- 前置：存在不同管理员姓名和账号的数据
- 步骤：分别以管理员姓名、登录账号搜索
- 期望：
  - 两类关键字都可命中对应企业
  - 前端列表与分页数据同步刷新

### TC-ENT-003 企业类型和状态筛选

- 步骤：
  1. 选择企业类型
  2. 选择状态
  3. 触发搜索
- 期望：
  - 列表结果符合筛选条件
  - 重置后恢复默认条件

## 2. 新增与编辑

### TC-ENT-004 新增企业成功并同步创建企业管理员

- 前置：当前用户具备平台超管权限
- 步骤：
  1. 打开新增企业弹窗
  2. 输入企业名称、企业类型、管理员姓名、登录账号、初始密码
  3. 提交保存
- 期望：
  - 企业保存成功
  - 同步创建企业管理员账号
  - 列表出现新企业
  - 人员管理中可查询到对应企业管理员

### TC-ENT-005 企业编码由后端生成

- 步骤：新增企业成功后查看详情或列表
- 期望：
  - 企业编码自动生成
  - 前端不要求用户手工输入企业编码
  - 若界面中展示企业编码字段，该字段为只读 / 回显语义

### TC-ENT-006 编辑企业且密码留空

- 前置：已有企业与企业管理员
- 步骤：
  1. 打开编辑弹窗
  2. 修改企业名称或管理员姓名
  3. 密码字段留空
  4. 提交保存
- 期望：
  - 非密码字段更新成功
  - 管理员密码不被重置

### TC-ENT-007 编辑企业并重置管理员密码

- 前置：已有企业与企业管理员
- 步骤：
  1. 打开编辑弹窗
  2. 输入新密码
  3. 提交保存
- 期望：
  - 企业信息更新成功
  - 管理员密码被重置

## 3. 删除守卫

### TC-ENT-008 企业下仍有有效非管理员人员时禁止删除

- 前置：目标企业下存在有效的非管理员人员账号
- 步骤：尝试删除该企业
- 期望：
  - 删除被阻止
  - 返回明确业务提示
  - 企业数据保持不变

### TC-ENT-009 企业下仅剩管理员时允许删除

- 前置：目标企业下不存在有效的非管理员人员账号
- 步骤：删除企业
- 期望：
  - 删除成功
  - 企业列表不再显示该企业
  - 不留下仍指向已删除企业的有效关联

## 4. 权限与租户隔离

### TC-ENT-010 新增 / 删除按钮仅平台超管可见

- 前置：分别使用平台超管与非平台超管登录
- 步骤：进入企业管理页
- 期望：
  - 平台超管可见新增、删除
  - 非平台超管不可见新增、删除

### TC-ENT-011 编辑权限受按钮码控制

- 前置：当前用户缺少 `enterprise_edit` / `tenant_edit`
- 步骤：进入企业管理页
- 期望：
  - 编辑入口隐藏或禁用
  - 越权调用后端时仍被拒绝

### TC-ENT-012 查询租户越权拦截

- 前置：存在其他租户企业数据
- 步骤：当前租户用户发起企业分页查询
- 期望：
  - 结果中不出现其他租户数据

### TC-ENT-013 写入租户越权拦截

- 前置：构造指向其他租户企业的数据提交
- 步骤：尝试编辑或删除其他租户企业
- 期望：
  - 后端拒绝写入
  - 返回稳定越权语义

## 5. 页面交互与错误反馈

### TC-ENT-014 写操作显示整体阻断式 loading

- 步骤：执行新增、编辑、删除任一写操作
- 期望：
  - 页面 / 弹窗出现整体阻断式 loading
  - 提交期间禁止重复点击与关闭

### TC-ENT-015 保存失败时保留输入

- 步骤：构造保存失败场景
- 期望：
  - 弹窗不被错误关闭
  - 已输入内容仍保留
  - 顶部显示错误反馈

### TC-ENT-016 后端业务文案透传

- 步骤：触发删除守卫或越权错误
- 期望：
  - 页面优先展示后端返回的业务文案
  - 不被统一替换成模糊的“操作失败”
