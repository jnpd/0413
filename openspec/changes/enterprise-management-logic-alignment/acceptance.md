# Enterprise Management Logic Alignment Acceptance

## 1. 规格验收

- enterprise 管理在 canonical openspec 中不再被定义为“占位页”
- 新规格明确冻结企业管理真实能力、权限、租户隔离和删除守卫
- 新规格明确写入“界面保持不变，只修逻辑”的约束

## 2. 业务验收

- 企业管理被定义为“企业 + 企业管理员”一体化页面
- 新增企业必须同步创建企业管理员
- 企业编码由后端生成，不允许手工编辑
- 编辑时密码留空表示不重置
- 删除前必须检查是否仍有有效非管理员人员账号

## 3. 权限与隔离验收

- 新增 / 删除仅平台超管可见
- 编辑 / 删除 / 查询都受按钮权限与后端越权校验约束
- 企业查询与写入都以后端租户上下文为准

## 4. 闭环验收

- enterprise 规格同时覆盖数据层、后端链路、前端链路与联动检查点
- 人员管理、角色映射、管理员关系被纳入同一 change 的影响面
- 文档中明确记录了剩余待确认风险，不把未确认点伪装成已验证事实
