# React Auth RBAC Foundation Test Cases

## 1. 登录与会话

### TC-LOGIN-001 登录成功

- 前置：后端 `8010` 可访问，测试账号有效
- 步骤：
  1. 输入账号密码
  2. 前端按约定加密密码并请求 `oauth/token`
  3. 登录成功后拉取 `user-info`
- 期望：
  - 返回 `access_token`
  - 本地保存 token 与 refresh token
  - 页面跳转到首个可访问路由

### TC-LOGIN-002 账号或密码错误

- 步骤：输入错误凭据提交
- 期望：
  - 登录失败
  - 显示后端可读错误文案
  - 不写入本地 token

### TC-LOGIN-003 刷新后恢复会话

- 前置：已登录
- 步骤：刷新浏览器
- 期望：
  - 本地 token 被读取
  - 自动拉取用户、菜单、按钮权限
  - 仍停留在可访问的受保护路由

### TC-LOGIN-004 401 刷新重放

- 前置：请求时 access token 失效但 refresh token 有效
- 步骤：访问受保护接口
- 期望：
  - 只触发一次 refresh
  - 原失败请求被自动重放
  - 页面不出现多次重复失败

### TC-LOGIN-005 refresh 失效

- 前置：access / refresh 均失效
- 步骤：访问受保护接口
- 期望：
  - 清空本地会话
  - 回到登录页
  - 给出统一失效提示

## 2. 菜单与路由

### TC-MENU-001 顶部菜单正常返回

- 步骤：启动 bootstrap
- 期望：
  - 拉取 `top-menu`
  - 根据活动 `topMenuId` 再拉取 `routes`
  - 侧栏按后端菜单生成

### TC-MENU-002 顶部菜单为空

- 步骤：`top-menu` 返回空列表
- 期望：
  - 前端继续尝试空 `topMenuId` 的 `routes`
  - 若仍无菜单，显示明确空导航状态

### TC-MENU-003 菜单项无对应 React 页面

- 步骤：后端返回一个未注册页面
- 期望：
  - 菜单可见
  - 点击后进入 placeholder 页面
  - 不静默丢弃该菜单

### TC-MENU-004 直接访问受保护路由

- 前置：已登录
- 步骤：浏览器输入某个已授权路由地址
- 期望：
  - bootstrap 后仍可进入该页面
  - 不被错误重定向

### TC-MENU-005 无页面权限

- 前置：当前用户不具备该页面菜单权限
- 步骤：访问该路由
- 期望：
  - 进入无权限页或统一提示
  - 不显示页面主体

## 3. 按钮权限

### TC-PERM-001 有按钮权限

- 前置：`buttons` 返回包含目标 code
- 步骤：进入页面
- 期望：按钮显示且可点击

### TC-PERM-002 无按钮权限

- 前置：`buttons` 不包含目标 code
- 步骤：进入页面
- 期望：按钮隐藏或禁用，行为保持一致
