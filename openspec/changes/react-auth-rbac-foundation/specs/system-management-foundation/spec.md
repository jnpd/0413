# 系统管理基础规格

## 状态

草稿，已可进入实现规划。

## 需求

### 需求 1. 登录鉴权必须走真实链路

前端必须对接 BladeX 真实鉴权接口，不能继续使用本地 mock 登录流程。

#### 场景：用户登录成功

- 前提：后端服务可访问
- 当：用户提交账号和密码
- 则：前端使用配置好的 SM2 公钥加密密码
- 且：请求 `POST /blade-auth/oauth/token`
- 且：持久化返回的 token 数据
- 且：完成用户信息、菜单和按钮权限的启动拉取

#### 场景：会话过期

- 前提：某个已鉴权请求返回 `401`
- 当：refresh token 仍然有效
- 则：前端执行单次刷新流程
- 且：自动重放失败请求

### 需求 2. 路由必须成为页面真源

前端页面导航必须基于真实路由，不能继续依赖组件本地状态切页。

#### 场景：刷新后应用重新装配

- 前提：本地已存在可用会话
- 当：应用刷新重载
- 则：受保护路由从启动状态中恢复

### 需求 3. 导航必须来自后端菜单

侧边栏必须由后端菜单数据经过适配后生成。

#### 场景：菜单项没有本地页面映射

- 前提：后端返回某个菜单项，但当前 React 侧没有注册对应页面
- 当：用户打开该菜单
- 则：前端路由到占位页
- 且：把该路径记录为已知缺口

### 需求 4. 按钮显示必须遵守权限码

前端动作按钮的显示或禁用必须由后端按钮权限码决定。

#### 场景：用户缺少动作权限码

- 前提：当前权限码集合中不包含目标动作
- 当：页面渲染
- 则：该按钮保持隐藏或禁用，且行为一致

### 需求 5. 系统管理必须按业务域拆分

系统管理页必须按业务语义拆分，而不是继续维持单文件混合实现。

#### 场景：人员管理加载

- 前提：操作人打开人员管理页
- 当：页面初始化
- 则：前端请求 `/blade-system/user/page`
- 且：渲染加载态、空态、错误态和成功态

#### 场景：角色管理加载

- 前提：操作人打开角色管理页
- 当：页面初始化
- 则：前端请求 `/blade-system/role/list`
- 且：使用返回的树结构，而不是继续消费本地 mock 角色数据

#### 场景：企业管理当前仍按后端缺口处理

- 前提：当前工作区尚未定位到企业管理扩展接口
- 当：操作人打开账户管理页
- 则：页面显示明确的占位说明和缺口提示
- 且：不得伪造企业管理已经成功联调
