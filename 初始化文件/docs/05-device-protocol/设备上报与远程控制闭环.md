## 1. 当前可确认范围

当前工作区尚未发现完整设备协议正文或 MQ 消费者实现，因此本页只冻结已验证到的链路骨架。

## 2. 上报侧已验证锚点

### 2.1 中间件配置

- Exchange：`exchange_arctech`
- 上报队列：`queue_report`
- 稠密采集队列：`queue_dense_collect`

### 2.2 落表目标

最新上报结果当前至少预留到 `tb_meter_latest`：

- 最新流量
- 阀门状态
- 电池电压
- 主副 IP
- ICCID
- 信号强度
- 最近上报时间
- 原始业务对象与原始报文

## 3. 远程控制侧已验证锚点

- 下发 routing key：`send_cmd`
- 超时 routing key：`send_cmd_timeout`

## 4. 项目级规则

- 设备最新态与历史态必须分开管理
- 原始报文、回执、超时都属于高风险链路，不允许只保留 happy path
- 设备上报链路的异常、重试、超时需要有可回查依据

## 5. 当前缺口

- 具体报文格式
- 消费者实现位置
- 回执结构
- 超时判定与重试策略
- 原始报文脱敏与存储边界

## 6. 测试建议

- 有上报时 `tb_meter_latest` 能刷新
- 上报异常时有可定位痕迹
- 下发超时时有独立通道或状态
- 回执与超时不混淆
