## 2. 当前工作区定位

当前工作区更接近“知识底座 + 后端实现样板 + 前端原型稿”，而不是一份完整可直接上线的单体工程交付包。

## 3. 当前可验证核心链路

### 3.1 登录与鉴权

- `POST /blade-auth/oauth/token`
- 业务请求头采用 `Authorization + Blade-Auth + Blade-Requested-With` 组合

### 3.2 仓库查询与写入

- 仓库统计、分页、详情、提交、删除、导入、出库接口已存在骨架
- 仓库主表为 `tb_device`
- 实时快照表为 `tb_meter_latest`

### 3.3 设备上报与命令预留

- RabbitMQ exchange、queue、routing key 已存在配置
- `tb_meter_latest` 说明上报数据至少有落表层设计

- 前端虽然已进入当前工作区，但目前仍是原型工程，尚未落到 Ant Design 后台模板
- 多租户、删除守卫、导入、重复出库等高风险逻辑尚未完全闭环
- 设备协议正文、MQ 消费者与回执处理代码未在当前工作区定位到
