# 交互行为契约

## 1. 目的

统一前端交互要求，避免不同页面交互风格漂移。

## 2. 固定交互点

- 页面入口和出口
- 查询和重置
- 列表操作
- 弹窗开关
- 提交确认
- 危险操作确认
- 成功反馈
- 失败反馈

## 3. 固定要求

- 每个页面必须定义入口和出口
- 查询区默认提供查询和重置
- 危险操作默认二次确认
- 成功操作默认可感知反馈
- 失败操作默认可读错误反馈

## 4. 弹窗和表单约定

- 打开时明确是新增、编辑还是查看
- 关闭时说明是否清空草稿
- 提交中必须禁用重复提交
- 取消时不应误提交

## 5. 列表页约定

- 搜索条件变化后，默认回到第一页
- 成功新增或编辑后，默认刷新列表
- 删除成功后，默认刷新当前列表

## 6. AI 使用要求

- 只要生成后台页面，默认先套这份交互契约
- 没有明确说明时，不允许 AI 自己发明页面级特殊交互
