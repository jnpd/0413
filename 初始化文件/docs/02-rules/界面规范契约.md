# 界面规范契约

## 1. 目的

统一后台管理类界面的基础规范，避免页面布局和文案风格混乱。

> 详细的一屏展示、自适应高度、表格滚动、弹窗尺寸、颜色和字体约束，参考 [前端实现规范.md](./前端实现规范.md)。本页保留为高层契约，不重复展开尺寸级细节。

## 2. 标准页面类型

- 列表页
- 详情页
- 表单页
- 弹窗页

## 3. 列表页基础布局

- 顶部为查询区
- 中部为操作区和表格区
- 底部为分页区

## 4. 固定规范

- 搜索项标签命名要统一
- 占位文案尽量短且直接
- 必填项明确标识
- 错误提示用用户能读懂的话
- 危险按钮与普通按钮风格明显区分

## 5. 一致性要求

- 同类页面使用同类布局
- 同类操作使用同类文案
- 同类状态使用同类颜色和标签
- 同类错误使用同类提示方式

## 6. AI 使用要求

- 后台页面默认先套列表页、详情页、表单页这三类规范
- 没有项目级设计系统时，不允许 AI 每页换一套布局语言
