# 接口联调契约

## 1. 目的

统一前后端接口联调和前端接口集成的工作方式。

## 2. 固定流程

1. 先冻结 `api-contract`
2. 后端自测通过
3. 提供请求和响应示例
4. 前端先接查询接口
5. 再接提交接口
6. 再补成功和失败反馈
7. 联调结论回写文档

## 3. 固定前置条件

- 契约已冻结
- 环境可访问
- 测试账号可用
- 测试数据可用
- 错误码稳定
- 返回结构稳定

## 4. 固定检查项

- 请求参数是否符合契约
- 响应字段是否符合契约
- 字段类型是否符合契约
- 枚举值是否符合契约
- 权限失败是否正确阻断
- 页面反馈是否正确

## 5. AI 使用要求

- 只要涉及前后端对接，默认引用这份契约
- 没有真实联调结果时，不允许 AI 宣称接口已接通
