# 业务规则选路索引

## 1. 目的

给人和 Agent 一个统一入口：当需求、缺陷、代码修改涉及某类问题时，先查本索引，再进入对应规则页，而不是在 `02-rules/` 下盲找。

## 2. 使用原则

- 先读 `README.md` 确认分层与优先级，再根据“当前问题类型”选规则页
- 一次需求可能命中多篇规则，按主规则 + 关联规则组合阅读
- 如果本索引里没有对应规则，先记录到 `docs/00-meta/缺口台账.md`，不要自己发明规则
- `前端实现规范.md`、`BladeX前端登录与仓库接口联调说明.md` 属于实施说明层，只能在上层规则命中后再联读

## 4. 阅读顺序建议

### 4.1 改接口

1. `接口规范.md`
2. `统一响应契约.md`
3. `接口联调契约.md`

### 4.2 改字段/状态

1. `状态枚举契约.md`
2. `状态一致性规则.md`
3. `枚举与字典代码落地规范.md`
4. `docs/00-meta/交付护栏契约.md`

### 4.3 改页面

1. `界面规范契约.md`
2. `交互行为契约.md`
3. `页面状态契约.md`
4. `前端实现规范.md`
5. 如有表单，再读 `表单提交契约.md`

### 4.4 改权限/隔离

1. `权限边界规则.md`
2. `权限控制契约.md`
3. `多租户隔离规则.md`

## 5. 给 Agent 的最小要求

- 触及 `02-rules` 时，优先先读本索引
- 回复里应说明“本次命中了哪些规则页”
- 如果某个规则页不存在或与当前目录结构不一致，先报冲突，不要跳过
