## 3. 实时列表的读取依赖

当前批量测试页并不只读本模块主表，还联动：

| 表名 | 作用 |
| --- | --- |
| `tb_device` | 设备基础信息、表号回写 |
| `tb_meter_latest` | 最新上报数据 |
| `tb_device_command_state` | 当前 IP、端口、上报周期、设备时间等命令状态快照 |

说明：

- `tb_device_command_state` 由命令闭环初始化器和命令协同器维护
- 实时页的“服务器 IP / 端口”“上报周期”等字段来自这类联表结果

## 4. 当前代码规则

### 4.1 设备加入批次

加入时会：

- 向 `tr_batch_detail_device` 新增一条当前态记录
- 把 `tb_device.prep_id` 回写成当前备货单号

当前只支持通过 `IMEI` 识别设备。

### 4.2 设备更新

更新接口只允许修改：

- `meter_no`
- `preset_base_value`
- `remark`

如果修改 `meter_no`，会同步更新：

- `tb_device.meter_no`

当前 `meter_no` 规则：

- 接口接收 1-14 位纯数字
- 保存前统一左补零到 14 位
- 重号校验按 `LPAD(meter_no, 14, '0')` 口径处理

### 4.3 人工判定

批量合格 / 不合格会更新：

- `pass_status`
- `test_status`
- `manual_pass_by`
- `manual_pass_by_name`
- `manual_pass_time`

### 4.4 命令动作

当前后端闭环已覆盖的动作码：

- `REPORT_DATA`
- `OPEN_VALVE`
- `CLOSE_VALVE`
- `WRITE_BASE_VALUE`
- `WRITE_METER_NO`
- `READ_REPORT_CYCLE`
- `SET_REPORT_CYCLE`
- `WRITE_METER_TIME`
- `WRITE_IP`
- `READ_IP`

当前未纳入闭环的动作：

- `UPGRADE_PROGRAM`

其中 `WRITE_METER_NO` 当前额外约束：

- 请求参数里的 `newMeterId` 会先按 14 位规则补零
- 写表成功后的回写、去重和后续下发都使用补零后的值

### 4.5 保存历史

保存历史时：

- 先写 `tl_batch_snapshot`
- 再把当前批次实时记录投影写入 `tr_batch_detail_history`

历史明细中保留的主要字段包括：

- `meter_no`
- `imei`
- `preset_base_value`
- `test_status`
- `pass_status`
- `remark`
- `report_time`
- `battery_voltage`
- `csq`
- `rsrp`
- `rsrq`
- `valve_status`
- `forward_total_flow`
- `reverse_total_flow`
- `main_ip`
- `backup_ip`
- `has_bench_data`

## 5. 关联关系

- `tr_batch_detail_device.batch_id -> tl_batch.batch_id`
- `tr_batch_detail_device.device_id -> tb_device.device_id`
- `tl_config_param.batch_id -> tl_batch.batch_id`
- `tl_batch_action_task.batch_id -> tl_batch.batch_id`
- `tr_batch_action_device.task_id -> tl_batch_action_task.task_id`
- `tl_batch_snapshot.batch_id -> tl_batch.batch_id`
- `tr_batch_detail_history.snapshot_id -> tl_batch_snapshot.snapshot_id`

## 6. 当前与旧文档的修正点

旧文档只写了“实时测试 + 配置参数表”，但当前代码已经形成完整动作闭环，实际至少包含：

- 当前态
- 阈值配置
- 动作任务头
- 动作设备明细
- 历史快照头
- 历史快照明细
- 命令状态快照读表

因此本模块数据库设计必须按闭环结构理解，而不能只看两张表。
