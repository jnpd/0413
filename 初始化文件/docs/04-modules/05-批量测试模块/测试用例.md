# 05 批量测试模块测试用例

## 1. 适用范围

覆盖实时列表、设备入批、批量动作、阈值配置、人工判定、保存历史与包装联动约束。

## 2. 用例清单

| 编号 | 类型 | 场景 | 预期结果 |
|---|---|---|---|
| `BTEST-01` | smoke | 未选择测试批次直接进入页面 | 页面要求先选定批次；未选定前不能继续核心操作 |
| `BTEST-02` | 主链路 | 按备货单号、关键字、判定状态筛选；关键字输入纯数字表号 | 列表正确筛选；1-14 位纯数字表号查询前自动补零到 14 位 |
| `BTEST-03` | 主链路 | 通过 15 位 `IMEI` 新增设备到当前批次 | 新增成功；同设备在当前批次内不可重复加入；若已绑定其他备货单号则拒绝 |
| `BTEST-04` | 边界 | 对缺少表号或通讯协议的设备执行开阀/关阀 | 前端直接禁用并提示原因，不误下发命令 |
| `BTEST-05` | 主链路 | 保存异常判定参数 | 电池电压、CSQ、无磁信号阈值保存成功，只影响列表标红，不直接改变合格状态 |
| `BTEST-06` | 主链路 | 写表号或更新设备表号 | 仅允许 1-14 位纯数字；自动补零；校验当前批次与仓库设备唯一性；成功后回写 `tb_device.meter_no` |
| `BTEST-07` | 主链路 | 批量合格/不合格人工判定 | 当前批次状态更新成功，并同步回写 `tb_device.test_status` |
| `BTEST-08` | 边界 | 尝试移除已进入包装流程的设备 | 移除被阻断，返回明确业务文案 |
| `BTEST-09` | 主链路 | 查看任务详情与单设备原始数据 | 任务详情展示成功/失败/超时汇总与逐台结果；原始数据展示源、时间、方向、报文内容 |
| `BTEST-10` | 回归 | 执行“保存历史” | 以固定备注 `页面保存历史` 保存快照；成功提示批次号与设备总数 |

## 3. 高风险回归触发

- `device/add`、`device/update`、`action/submit`、`snapshot/save` 接口变更
- 表号补零规则变更
- 人工判定与 `tb_device.test_status` 同步规则变更
- 包装联动删除守卫变更
