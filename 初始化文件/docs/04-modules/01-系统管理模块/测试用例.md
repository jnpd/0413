# 01 系统管理模块测试用例

## 1. 适用范围

覆盖登录、公共框架、公告管理、企业管理、人员管理、角色管理与表格列设置。

## 2. 用例清单

| 编号 | 类型 | 场景 | 预期结果 |
|---|---|---|---|
| `SYS-01` | smoke | 使用正确账号密码登录；若开启验证码/租户识别，则同时走验证码与租户识别链路 | 成功获取 `token`、`user-info`，进入默认首页，错误提示不残留 |
| `SYS-02` | 边界 | 账号、密码或验证码错误后重试登录 | 登录被拒绝，页面保留可读错误原因，不误进入系统 |
| `SYS-03` | smoke | 登录后加载公共框架 | 左侧动态菜单、顶部模块切换、按钮权限、滚动公告分别按接口返回正常渲染 |
| `SYS-04` | 主链路 | 新增或编辑公告 | 标题与正文保存成功；`summary` 自动生成；类型与优先级沿用后端默认值 |
| `SYS-05` | 权限 | 平台超管新增企业 | 企业保存成功；企业编码由后端生成；同步创建企业管理员账号 |
| `SYS-06` | 边界 | 删除仍存在有效非管理员人员账号的企业 | 删除被阻断，并返回明确业务文案 |
| `SYS-07` | 主链路 | 非平台超管新增或编辑人员 | 所属企业被锁定为当前登录人所属企业；企业管理员角色按企业类型自动解析 |
| `SYS-08` | 权限 | 尝试编辑或删除“主管理员” | 页面不允许操作或后端拒绝操作，不能误修改主管理员 |
| `SYS-09` | 主链路 | 新增角色后进入菜单权限配置 | 角色可正常保存；菜单树读取正常；保存授权时只提交菜单权限 |
| `SYS-10` | 回归 | 角色列表按关键字、状态、企业过滤 | 列表与分页结果直接使用后端口径，不再回退旧树状拍平逻辑 |
| `SYS-11` | 回归 | 发布公告、保存企业、保存人员、保存角色、授权、删除等写操作 | 提交中显示整体界面阻断式 `loading`，不能重复提交或误关弹窗 |
| `SYS-12` | 回归 | 修改表格列顺序后刷新页面 | 按页面路径和表格类型恢复用户级列设置 |

## 3. 高风险回归触发

- 登录与鉴权链路变更
- 菜单、按钮权限、顶部公告接口变更
- 企业删除守卫变更
- 人员角色与企业隔离逻辑变更
- 角色授权接口或权限树结构变更
