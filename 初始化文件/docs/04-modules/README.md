# 04 数据模型与真源归档

## 1. 目的

`docs/04-modules/` 同时承担两层职责：

- 项目级摘要：用 [核心实体说明.md](./核心实体说明.md) 快速说明当前仓库已验证的数据实体、字段和状态
- 真源归档：归档外部项目中的总 PRD、交互说明、总体数据库设计，以及各模块拆解后的 PRD / 数据库设计
- 模块测试库：在各模块目录下补 `测试用例.md`，把当前 PRD、数据库设计和项目规则提炼成可执行测试场景

## 2. 阅读顺序

### 2.1 先看摘要

- [核心实体说明.md](./核心实体说明.md)

适合：

- 快速理解当前仓库已验证实体
- 先看当前系统的字段、状态和风险点

### 2.2 再看总资料

- [00-总资料/总PRD.md](./00-总资料/总PRD.md)
- [00-总资料/交互说明.md](./00-总资料/交互说明.md)
- [00-总资料/总体数据库设计.md](./00-总资料/总体数据库设计.md)

适合：

- 回看完整产品边界
- 查看系统交互方式和 MQ 队列定义
- 阅读完整数据库设计原文

### 2.3 最后按模块查看

- [01-系统管理模块](./01-系统管理模块/README.md)
- [02-表具档案模块](./02-表具档案模块/README.md)
- [03-仓库管理模块](./03-仓库管理模块/README.md)
- [04-批次管理模块](./04-批次管理模块/README.md)
- [05-批量测试模块](./05-批量测试模块/README.md)
- [06-测试历史模块](./06-测试历史模块/README.md)
- [07-包装管理模块](./07-包装管理模块/README.md)

每个模块默认阅读顺序：

1. `模块PRD.md`
2. `数据库设计.md`
3. `测试用例.md`
