# 07 包装管理模块测试用例

## 1. 适用范围

覆盖包装任务创建、装箱扫码、容量与同箱约束、下一箱流程、编辑箱与打印口径。

## 2. 用例清单

| 编号 | 类型 | 场景 | 预期结果 |
|---|---|---|---|
| `PKG-01` | smoke | 进入包装列表并按表号、箱号、备货单号搜索 | 三级层级表格查询正常；搜索结果按“表号 -> 箱号 -> 备货单号”链路命中 |
| `PKG-02` | 主链路 | 新增包装记录并创建/复用包装任务草稿 | 备货单号、表具档案、检验员、预览箱号、包装进度正常回显 |
| `PKG-03` | 主链路 | 扫描合法表号入箱 | `meterNo` 仅接受 1-14 位纯数字；自动补零；箱内展示统一按 14 位口径 |
| `PKG-04` | 边界 | 平台设备不满足备货单号匹配、`testStatus = 1` 或已被其他箱包装 | 扫码校验被拒绝，并返回明确业务原因 |
| `PKG-05` | 主链路 | 找不到平台设备，但已选定表具档案时扫描入箱 | 允许按“非平台设备”入箱，不强制要求平台设备存在 |
| `PKG-06` | 边界 | 单箱连续扫描超过 10 个表号或混入不同 `meterId` 设备 | 超过 `BOX_CAPACITY = 10` 或同箱档案不一致时被阻断 |
| `PKG-07` | 主链路 | 点击“下一箱” | 先保存当前箱，再刷新列表、切换下一箱草稿、清空扫码表号，并尝试本地打印当前箱标签 |
| `PKG-08` | 回归 | 编辑箱内设备并保存修改 | 页面通过整箱 `update` 重写关系；删除单个表号不直接调用 `device/remove` |
| `PKG-09` | 回归 | 顶部打印与箱级打印 | 当前均走前端本地打印 helper，不调用后端 `box/print` |
| `PKG-10` | 回归 | 保存包装记录、下一箱等写操作提交中 | 页面与装箱弹窗显示整体阻断式 `loading`，不能继续扫码、切箱或重复提交 |

## 3. 高风险回归触发

- 装箱校验规则变更
- 平台设备/非平台设备混合策略变更
- `box/save`、`box/update`、`box/next` 接口变更
- 打印实现由前端本地 helper 切换到后端接口
