# 03 仓库管理模块测试用例

## 1. 适用范围

覆盖仓库分页查询、统计卡片、入库、预入库、正式导入、删除守卫与状态口径。

## 2. 用例清单

| 编号 | 类型 | 场景 | 预期结果 |
|---|---|---|---|
| `WMS-01` | smoke | 加载统计卡片与分页列表，并按关键字、库存状态、测试状态筛选 | `stat`、`page` 返回正常；列表筛选与统计口径一致 |
| `WMS-02` | 主链路 | 手动入库一台合法设备 | 保存成功；新设备默认 `stockStatus = 0`、`testStatus = 0` |
| `WMS-03` | 主链路 | 扫码入库成功后连续扫描下一台设备 | 与手动入库共用提交接口；保存后清空表单并保持弹窗打开 |
| `WMS-04` | 边界 | 提交非法 `IMEI`、缺失表具档案、缺失通讯协议或非法表号 | 前后端校验一致；`IMEI` 必须为 15 位数字；表号按 14 位补零 |
| `WMS-05` | 主链路 | 预入库文件全部成功导入 | 当前弹窗自动关闭，顶部 `Toast` 提示成功条数，不写入 `tb_device` |
| `WMS-06` | 边界 | 预入库文件存在失败行 | 当前弹窗保留，展示成功/失败摘要与逐条失败原因，禁止误关闭上传中的弹窗 |
| `WMS-07` | 主链路 | 正式批量导入仓库设备 | 页面上传成功并展示导入结果摘要，不要求页面提供导出模板入口 |
| `WMS-08` | 边界 | 删除已出库、已进入测试批次、已生成历史快照或已进入包装流程的设备 | 删除被阻断，并返回稳定错误语义 |
| `WMS-09` | 回归 | 同租户存在已软删同 `IMEI` 设备后再次入库 | 走“恢复设备”逻辑，不新增重复记录 |
| `WMS-10` | 回归 | 仓库写操作提交中 | 入库、导入等写操作显示整体阻断式 `loading`，不能重复提交 |

## 3. 高风险回归触发

- `submit/remove/import/outbound` 接口变更
- 表号补零口径变更
- 预入库模板列或失败摘要结构变更
- 删除守卫依赖关系变更
