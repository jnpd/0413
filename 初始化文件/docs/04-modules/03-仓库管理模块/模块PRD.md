# 仓库管理模块 PRD

## 1. 模块定位

当前仓库管理模块对应：

- 前端组件：`frontEnd/src/views/product/warehouse/index.tsx`
- 后端接口前缀：`/blade-wms/warehouse`

它负责：

- 仓库设备台账查询
- 手动入库
- 扫码入库
- 预入库标识池导入
- 批量导入正式仓库设备

## 2. 当前页面能力

### 2.1 搜索与筛选

当前页面支持：

- 关键字搜索：`IMEI / 水表表号 / 备货单号`
- 库存状态筛选
- 测试状态筛选

### 2.2 顶部统计卡片

当前统计卡片只有 4 项：

- 在库设备
- 已出库
- 待测
- 不合格

### 2.3 顶部操作

当前页面操作按钮：

- 手动入库
- 扫码入库
- 预入库
- 批量导入
- 批量出库

其中真实状态如下：

- 手动入库：已接入
- 扫码入库：已接入
- 预入库：已接入
- 批量导入：已接入
- 批量出库：按钮存在，但当前页面没有选择器，按钮为禁用状态

### 2.4 列表字段

当前列表字段为：

- IMEI
- 水表表号
- 入库时间
- 正向累积流量
- 反向累积流量
- 备货单号
- 主/副 IP
- 表具名称
- 表具型号
- 口径
- 通讯协议
- 运营商
- 阀门状态
- 电池电压
- 装箱号
- 厂商代码
- ICCID
- CSQ / RSRP / RSRQ
- 库存状态
- 测试状态
- 出库时间
- 出库操作人员
- 备注
- 更新时间
- 操作

## 3. 当前表单与导入能力

### 3.1 手动入库 / 扫码入库

当前表单字段：

- 表具档案
- 备货单号（可空）
- 水表表号（可空）
- IMEI
- 通讯协议
- 运营商（可空）
- 厂商代码（可空）
- 备注（可空）

校验规则：

- IMEI 必须是 15 位数字
- 表具档案必选
- 通讯协议必选
- 水表表号如填写，必须是 1-14 位纯数字；少于 14 位会自动补零

扫码入库与手动入库共用同一提交接口，只是交互模式不同：

- 手动入库：保存后关闭弹窗
- 扫码入库：保存成功后清空表单并保持弹窗打开，方便连续扫码

### 3.2 预入库

当前“预入库”不是正式入库，而是把标识写入标识池。

模板列固定为：

- IMEI
- 来源批次
- 备注

预入库当前不会写入 `tb_device`。

当前预入库上传交互规则：

- 点击“上传文件”后，提交按钮进入 `loading`，文案切换为“上传中...”
- 上传进行中禁止关闭弹窗，也不允许重复提交
- 全部成功时自动关闭当前弹窗，并通过顶部 `Toast` 提示导入成功条数
- 若存在失败行，则保留当前弹窗并展示成功/失败摘要与逐条失败原因
- 模板列、上传按钮文案、结果 `Toast` 与已选文件文案由前端统一模型维护，避免页面散落硬编码

### 3.3 批量导入

当前正式仓库导入已接入，但页面只负责上传文件和展示导入结果摘要，不提供导出模板入口。

## 5. 当前代码规则

### 5.1 入库

- 新增时若发现同租户已有被软删除的同 IMEI 设备，会按“恢复设备”处理，而不是新插入一条
- 新建设备默认：
  - `stockStatus = 0`
  - `testStatus = 0`
- `meter_no` 会按 14 位口径入库；查询、唯一性校验和详情回显都按补零后的值处理
- 编辑历史脏数据时，如原表号未改动，可临时保留原值；一旦修改，必须满足新的 14 位补零规则

### 5.2 删除

删除会被以下情况拦截：

- 设备已出库
- 设备已进入测试批次
- 设备已生成历史快照
- 设备已进入包装流程

### 5.3 出库

后端支持出库，出库后会更新：

- `stockStatus = 1`
- `outboundTime`
- `outboundOperatorId`
- `outboundOperatorName`

但当前页面尚未提供批量勾选入口，所以前端用户暂时不能从该页完成出库。

## 6. 当前未接入或仅部分实现的内容

- 预入库与正式入库之间没有页面级“转正式入库”流程
- 页面没有导出功能
