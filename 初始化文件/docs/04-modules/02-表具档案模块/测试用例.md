# 02 表具档案模块测试用例

## 1. 适用范围

覆盖表具档案分页、搜索、排序、新增、编辑、删除与下游引用约束。

## 2. 用例清单

| 编号 | 类型 | 场景 | 预期结果 |
|---|---|---|---|
| `ARCH-01` | smoke | 进入列表页并按关键字搜索 | 可按表具名称、型号、口径、备注查询，搜索后返回第一页 |
| `ARCH-02` | 主链路 | 对支持排序的列执行排序 | 名称、型号、口径、备注、创建时间排序结果稳定，列表与分页一致 |
| `ARCH-03` | 主链路 | 新增一条合法表具档案 | 名称、型号、口径必填校验生效；保存成功后列表可见 |
| `ARCH-04` | 边界 | 同租户下新增重复 `meter_name` | 保存被拒绝，并返回唯一性错误文案 |
| `ARCH-05` | 主链路 | 编辑已有表具档案 | 允许更新名称、型号、口径、备注；更新后列表与下拉显示同步 |
| `ARCH-06` | 边界 | 删除已被 `tb_device` 引用的表具档案 | 删除被阻断，并返回明确业务文案 |
| `ARCH-07` | 边界 | 删除已被 `tl_package_box` 引用的表具档案 | 删除被阻断，不能破坏包装模块引用 |
| `ARCH-08` | 回归 | 获取下拉列表并在下游模块显示 | `displayName` 按 `meterName / meterModel / meterCaliber` 拼接展示 |
| `ARCH-09` | 回归 | 新增、编辑、删除写操作提交中 | 页面与弹窗进入整体阻断式 `loading`，不能重复提交 |

## 3. 高风险回归触发

- `meter_name` 唯一性规则变更
- 删除守卫引用表调整
- 下游仓库/包装模块使用表具下拉逻辑变更
