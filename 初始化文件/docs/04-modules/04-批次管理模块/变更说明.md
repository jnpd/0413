# 批次管理模块 Change 基线说明

> 该文件不再记录“未来计划”，而是记录当前批次管理模块已经落到代码里的基线能力和边界。

## 1. 当前前端现状

前端落点：

- `frontEnd/src/views/product/BatchManagement/index.tsx`

当前已落地能力：

- 备货单号分组列表
- 分组行展开/收起
- 新增备货单号
- 编辑备货单号
- 删除备货单号
- 新增测试批次
- 编辑测试批次
- 删除测试批次
- 点击“去测试”跳转批量测试页

当前页面没有：

- 导入导出
- 日期筛选
- 状态筛选
- 独立详情页

## 3. 已落地业务规则

### 3.1 编号规则

- 备货单号：`2603-001`
- 测试批号：`2603-001-P001`

### 3.2 新增批次

- 批次号由后端自动生成
- 前端不提供手输批次号的“新建模式”
- 新增成功后可直接跳到批量测试页

### 3.3 重命名备货单号

- 会同步重写同组所有测试批号前缀
- 若该备货单号已有历史快照，需要二次确认

### 3.4 删除约束

- 删除备货单号：组下不能还有有效测试批次
- 删除测试批次：不能已有历史快照、检验台绑定、包装记录

## 4. 前后端契约要点

### 4.1 备货单号

- 前端输入校验与后端校验一致：`^\d{4}-\d{3}$`

### 4.2 批次号

- 前端编辑校验与后端校验一致：`^{备货单号}-P\d{3}$`

### 4.3 删除按钮

列表里的删除按钮依赖后端返回的 `canDelete`，不是前端本地猜测。

## 5. 与其他模块衔接

- 仓库管理引用备货单号下拉
- 批量测试引用批次下拉
- 包装管理引用备货单号下拉

## 6. 当前文档修正点

旧版 Change 文档里把很多内容写成“待实施”。当前代码实际已经完成以下事项：

- 备货单号与批次号的双层列表
- 新增批次自动生成编号
- 重命名二次确认
- 删除约束联动历史/检验台/包装

因此本文件现以“当前代码基线”描述为准。
