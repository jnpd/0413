## 1. 测试策略原则

- 先主链，再边界
- 高风险规则必须有独立用例
- 没有 smoke 证据，不宣称完成
- 页面体验问题也要纳入 baseline，不只测接口通断

## 2. 全局必测场景

### 2.1 登录与鉴权

- 正确登录能获取 token
- 登录成功后能访问仓库接口
- 登录失效后不能继续误操作

### 2.2 多租户越权

- 查询越权
- 写操作越权

### 2.3 删除守卫

- 已被业务链路引用的数据不能直接删除
- 删除失败时返回明确业务文案

### 2.4 唯一约束

- IMEI 重复
- 写号后 `meter_no` 重复

### 2.5 出库一致性

- 出库后库存状态变化正确
- 重复出库被阻断

### 2.6 导入与文件上传

- 模板字段检查
- 失败反馈可读
- 回滚策略明确

### 2.7 设备链路

- 上报能落最新态
- 命令超时/回执有区分

## 3. 当前重点模块场景

### 3.1 仓库模块

- `stat` / `page` / `detail` 接口 smoke
- `submit` 基础校验
- `remove` 删除守卫
- `outbound` 状态一致性
- `import` 文件与失败反馈

### 3.2 历史快照与检验台

- 历史只读
- 绑定唯一
- 删除守卫
- 租户边界

## 4. 回归触发规则

以下任一改动都要回归本页：

- 登录与鉴权
- 多租户逻辑
- 仓库状态字段
- 导入/导出
- 设备上报、回执、超时链路
