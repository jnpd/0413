# 批次管理模块 PRD

## 1. 模块定位

当前批次管理模块对应：

- 前端组件：`frontEnd/src/views/product/BatchManagement/*`
- 后端接口前缀：`/blade-wms/batch-management`

该模块当前只做两类对象管理：

- 备货单号分组
- 测试批次号

## 1.1 提交交互约束

批次管理里的新增批次、重命名备货单号、删除备货单号、删除测试批次、保存弹窗表单，都必须显示整体界面阻断式 loading。

提交中要同时阻断当前页面和当前弹窗上下文，避免重复点击或在一次写操作未完成时继续改动同页数据。

## 2. 当前页面结构

### 2.1 列表结构

当前页面是“备货单号分组 + 批次明细”的可展开表格。

表头字段：

- 备货单号
- 批次号
- 设备总数
- 合格
- 异常
- 待测
- 备注
- 创建时间
- 操作

### 2.2 搜索

当前只支持一个搜索框：

- 搜索测试批号、备货单号

没有额外状态筛选、日期筛选或企业筛选。

## 3. 当前页面能力

### 3.1 备货单号分组行

分组行支持：

- 展开 / 收起明细
- 新增批次
- 编辑备货单号
- 删除备货单号

分组行显示的是该备货单号下所有有效批次的聚合统计。

### 3.2 批次明细行

批次行支持：

- 去测试
- 编辑批次
- 删除批次

### 3.3 新增/编辑弹窗

当前页面有两个弹窗：

1. 备货单号弹窗
   - 新增备货单号
   - 编辑备货单号

2. 批次弹窗
   - 编辑批次

当前字段：

- 备货单号
- 测试批号（仅编辑批次时）
- 备注

## 4. 当前代码规则

### 4.1 编码规则

备货单号格式：

- `2603-001`

测试批号格式：

- `2603-001-P001`

后端真实规则：

- 备货单号必须匹配 `^\d{4}-\d{3}$`
- 测试批号必须匹配 `^{备货单号}-P\d{3}$`

### 4.2 新增批次

当前“新增批次”不是手动填写批次号，而是：

- 先选中某个备货单号
- 调用 `POST /batch/create`
- 后端自动生成下一个批次号
- 成功后自动跳转到批量测试页

### 4.3 备货单号重命名

当前重命名规则：

- 该组下所有测试批号前缀同步改写
- 批次后缀 `P001 / P002 ...` 保持不变
- 如果已有历史快照，则先返回二次确认

### 4.4 删除约束

删除备货单号：

- 只要该备货单号下还有有效测试批次，就不允许删除

删除测试批次：

- 只要已产生历史快照，就不允许删除
- 只要已绑定检验台数据，就不允许删除
- 只要已产生包装记录，就不允许删除
- 包装记录优先按批次明细设备与包装关系表判断；若关系表不存在，则回退到设备主档中的包装字段

## 5. 当前接口

| 能力 | 接口 |
| --- | --- |
| 分页查询 | `GET /blade-wms/batch-management/page` |
| 新增备货单号 | `POST /blade-wms/batch-management/prep/submit` |
| 重命名备货单号 | `POST /blade-wms/batch-management/prep/rename` |
| 删除备货单号 | `POST /blade-wms/batch-management/prep/remove` |
| 新增测试批次 | `POST /blade-wms/batch-management/batch/create` |
| 编辑测试批次 | `POST /blade-wms/batch-management/batch/submit` |
| 删除测试批次 | `POST /blade-wms/batch-management/batch/remove` |
| 备货单号下拉 | `GET /blade-wms/batch-management/prep/select-list` |
| 批次下拉 | `GET /blade-wms/batch-management/batch/select-list` |

## 6. 与其他模块的关系

- 仓库管理入库时可选备货单号，数据来自 `prep/select-list`
- 批量测试当前批次选择来自 `batch/select-list`
- 包装管理新增包装记录时，备货单号下拉来自 `prep/select-list`

## 7. 当前未实现的内容

当前页面没有这些能力：

- 批次导出
- 批次导入
- 批次状态筛选
- 单独的批次详情页
