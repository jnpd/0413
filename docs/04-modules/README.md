## 2.4 测试用例字段约定

各模块 `测试用例.md` 统一使用以下字段，方便人和 AI 同时消费：

| 字段 | 含义 | 使用要求 |
|---|---|---|
| `编号` | 模块内稳定用例 ID | 固定前缀 + 顺序号，如 `SYS-01`、`WMS-03` |
| `优先级` | 回归优先顺序 | `P0` = 主链阻断 / 安全 / 高风险闭环；`P1` = 高频主链或关键边界；`P2` = 补充回归 |
| `类型` | 用例目的 | 推荐使用 `smoke / 主链路 / 边界 / 权限 / 回归` |
| `前置条件` | 执行前必须满足的上下文 | 账号、角色、批次、历史数据、依赖记录等写清，不要默认读者脑补 |
| `场景` | 本次验证动作 | 用一句话描述验证对象，必要时点明关键输入 |
| `预期结果` | 业务或界面应达到的结果 | 结果要可判定，避免写成空泛“功能正常” |
| `证据点` | 执行后要看什么来证明结果 | UI 文案、接口响应、数据库字段、日志 / traceId、文件结果等 |
| `关联接口/对象` | 本用例主要命中的接口、表、字段或状态 | 用于 AI 做 impact analysis 和回归选路 |

给 AI 的最低要求：

- 先按 `优先级` 决定回归顺序，默认 `P0 > P1 > P2`
- 先看 `前置条件` 再执行，不允许跳过上下文假设
- 回答测试结果时，应引用 `证据点`
- 涉及字段、状态、权限、删除守卫、多租户时，应结合 `关联接口/对象` 做闭环检查

推荐章节结构：

1. 适用范围
2. 字段说明
3. 用例清单
4. 建议执行顺序
5. 最小回归集
6. 高风险回归触发
7. 关联规则

其中：

- `建议执行顺序` 用于告诉人和 AI 先跑哪些用例、再补哪些用例
- `最小回归集` 用于用户只要求“快速验证”“先 smoke 一轮”时的默认集合
