# 包装管理模块 PRD

## 1. 模块定位

当前包装管理模块对应：

- 前端组件：`frontEnd/src/views/product/Packaging/*`
- 后端接口前缀：`/blade-wms/packaging`

模块职责是：

- 按备货单号创建包装任务
- 按箱号装箱
- 扫描水表表号入箱
- 查看三级层级包装清单
- 打印条码标签

## 1.1 提交交互约束

包装管理中的保存包装记录、下一箱等写操作，提交中必须显示整体界面阻断式 loading，而不是只给当前按钮加 loading。

整体阻断需要覆盖当前页面和装箱弹窗上下文，避免用户在一箱保存未完成时继续扫描、切箱或重复提交。

## 2. 当前页面结构

### 2.1 列表结构

当前列表是三级层级表格：

- L1：备货单号层
- L2：箱号层
- L3：单表号层

当前列表字段：

- 层级
- 备货单号
- 箱号
- 条码信息（条形码图片）
- 水表表号
- 表具名称
- 表具类型
- 表具口径
- 检验员
- 包装时间
- 来源
- 备注
- 操作

### 2.2 搜索

当前搜索提示和实际逻辑一致，支持：

- 表号 -> 箱号 -> 备货单号
- 箱号 -> 备货单号
- 搜索词或分页大小变化时先回到第一页，同一查询条件下只保留 1 次有效列表请求

## 3. 当前页面能力

### 3.1 顶部操作

当前顶部按钮：

- 新增包装记录
- 打印标签

说明：

- “新增包装记录”打开装箱弹窗
- “打印标签”当前是前端本地打印 helper，不是调用后端 `/box/print`

### 3.2 新增包装记录弹窗

当前弹窗字段：

- 备货单号
- 表具名称
- 表具类型（自动获取）
- 表具口径（自动获取）
- 检验员（系统生成）
- 预览箱号
- 包装进度
- 备注
- 预览条码信息
- 扫描水表表号

### 3.3 当前保存按钮

新增模式下有两个保存按钮：

- 下一箱
- 保存包装记录

编辑模式下只有：

- 保存修改

### 3.4 列表操作

当前箱级操作只有：

- 编辑
- 打印

当前列表没有单独的“删除箱”按钮。

## 4. 当前代码规则

### 4.1 箱容量

当前后端固定：

- `BOX_CAPACITY = 10`

即单箱最多 10 个表号。

### 4.2 表号规则

当前扫描和保存都要求：

- `meterNo` 必须为 1-14 位纯数字
- 少于 14 位自动补零
- 搜索结果、箱内列表和扫描回显统一按 14 位展示

### 4.3 同箱约束

当前同箱只允许同一 `meterId`：

- 已选择表具档案后，新加入设备必须表具档案一致

### 4.4 平台设备与非平台设备

当前后端真实行为：

- 如果能按补零后的 `meterNo` 在平台设备里找到记录，则按平台设备入箱
- 如果找不到平台设备，但页面已选定表具档案，则允许作为“非平台设备”入箱

因此旧文档里如果写成“包装只能使用平台设备”，需要以当前代码修正。

### 4.5 测试通过约束

平台设备要满足：

- 备货单号匹配
- `testStatus == 1`
- 未被其他箱包装

否则不允许入箱。

### 4.6 下一箱

当前“下一箱”真实行为：

1. 先保存当前箱
2. 刷新列表
3. 切换到下一箱草稿
4. 清空当前扫码表号
5. 前端尝试本地打印当前箱标签

## 5. 当前接口

| 能力 | 接口 |
| --- | --- |
| 分页查询 | `GET /blade-wms/packaging/page` |
| 创建/复用包装任务草稿 | `POST /blade-wms/packaging/task/create` |
| 扫码校验 | `POST /blade-wms/packaging/device/verify` |
| 保存当前箱 | `POST /blade-wms/packaging/box/save` |
| 编辑当前箱 | `POST /blade-wms/packaging/box/update` |
| 保存并下一箱 | `POST /blade-wms/packaging/box/next` |

后端已提供但当前前端页面未实际使用：

- `POST /blade-wms/packaging/box/print`
- `POST /blade-wms/packaging/device/remove`

## 6. 当前前端与后端接线差异

### 6.1 打印

当前页面的打印分两种：

- 顶部“打印标签”：前端本地批量打印
- 箱级“打印”：前端本地单箱打印

当前页面没有调用后端 `box/print`。

### 6.2 删除单个表号

后端有 `device/remove` 接口，但当前页面删除单个表号的方式是：

- 在编辑弹窗里先从本地列表移除
- 再通过整箱 `update` 提交重写关系
