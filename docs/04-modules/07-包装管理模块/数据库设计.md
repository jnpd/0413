## 3. 关键编码规则

### 3.1 包装任务号

当前后端生成规则：

- `PKG-yyyyMMdd-序号`

例如：

- `PKG-20260408-001`

### 3.2 箱号

当前后端生成规则：

- `{prepNo}-00001`

也就是以备货单号为前缀，加 5 位流水号。

## 4. 关键规则

### 4.1 容量

当前固定：

- `BOX_CAPACITY = 10`

### 4.2 表号与平台设备

`tr_package_box_device` 中：

- `device_id` 有值：平台设备
- `device_id` 为空：非平台设备
- `is_platform_device = 1/0` 对应上述两类

当前 `meter_no` 规则：

- 接口接收 1-14 位纯数字
- 保存前统一左补零到 14 位
- 平台设备匹配、箱内查找和搜索命中都按 `LPAD(meter_no, 14, '0')` 口径处理

### 4.3 设备回填

平台设备入箱后，后端会把以下字段回填到 `tb_device`：

- `package_id`
- `box_id`
- `box_no`
- `box_seq`

编辑箱内容或删除箱内设备时，也会同步更新回填结果。

## 5. 关联关系

- `tl_package.prep_id -> tl_stock_prep.prep_id`
- `tl_package_box.package_id -> tl_package.package_id`
- `tl_package_box.prep_id -> tl_stock_prep.prep_id`
- `tl_package_box.meter_id -> tb_meter.meter_id`
- `tr_package_box_device.box_id -> tl_package_box.box_id`
- `tr_package_box_device.package_id -> tl_package.package_id`
- `tr_package_box_device.device_id -> tb_device.device_id`（非平台设备可为空）

## 6. 当前代码约束

### 6.1 创建任务

同一 `prep_id` 下，后端会优先复用当前有效包装任务；不存在时再新建 `tl_package`。

### 6.2 扫码校验

校验时会检查：

- 表号格式必须为 1-14 位纯数字，少于 14 位自动补零
- 是否属于当前备货单号
- 是否测试合格
- 是否已包装
- 是否与当前箱表具档案兼容
- 当前箱是否已满

### 6.3 保存箱

保存时会：

1. 写 `tl_package_box`
2. 写 `tr_package_box_device`
3. 回填 `tb_device`
4. 更新 `tl_package.box_qty`

### 6.4 编辑箱

编辑当前不是对旧关系逐条修改，而是：

- 先软删旧关系
- 再按新的表号清单重建关系
- 再回填平台设备包装字段

### 6.5 删除单个表号

后端 `removeSavedDevice` 已支持：

- 从箱里删除 1 个表号
- 自动重排 `box_seq`
- 空箱时自动删箱

但当前前端页面没有直接调用该接口。

## 7. 与旧文档的修正点

旧文档把打印、删除、平台设备约束都写成单一路径；当前代码真实情况是：

- 打印接口存在，但前端主流程走本地打印 helper
- 平台设备不是唯一来源，允许非平台设备入箱
- 单表号删除接口存在，但前端当前走整箱更新

因此本文件按实际数据库使用方式重写。
