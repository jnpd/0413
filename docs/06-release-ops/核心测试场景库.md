## 1. 测试策略原则

- 先主链，再边界
- 高风险规则必须有独立用例
- 没有 smoke 证据，不宣称完成
- 页面体验问题也要纳入 baseline，不只测接口通断

## 2. 全局必测场景

### 2.1 登录与鉴权

- 正确登录能获取 token
- 登录成功后能访问仓库接口
- 登录失效后不能继续误操作

### 2.2 多租户越权

- 查询越权
- 写操作越权

### 2.3 删除守卫

- 已被业务链路引用的数据不能直接删除
- 删除失败时返回明确业务文案

### 2.4 唯一约束

- IMEI 重复
- 写号后 `meter_no` 重复

### 2.5 出库一致性

- 出库后库存状态变化正确
- 重复出库被阻断

### 2.6 导入与文件上传

- 模板字段检查
- 失败反馈可读
- 回滚策略明确

### 2.7 设备链路

- 上报能落最新态
- 命令超时/回执有区分

## 3. 当前重点模块场景

### 3.1 仓库模块

- `stat` / `page` / `detail` 接口 smoke
- `submit` 基础校验
- `remove` 删除守卫
- `outbound` 状态一致性
- `import` 文件与失败反馈

### 3.2 历史快照与检验台

- 历史只读
- 绑定唯一
- 删除守卫
- 租户边界

### 4.2 按变更类型看

| 变更类型 | 优先回归模块 | 说明 |
|---|---|---|
| 登录、鉴权、菜单、角色权限 | 01 系统管理 | 先看登录主链、菜单与按钮权限、角色授权 |
| 表具档案字段、下拉、删除守卫 | 02 表具档案、03 仓库管理、07 包装管理 | 档案变化通常会影响入库和包装下游 |
| 仓库字段、入库、导入、出库、IMEI / 表号口径 | 03 仓库管理、04 批次管理、05 批量测试、07 包装管理 | 仓库是下游多个模块的源头 |
| 备货单号、批次号、批次删除守卫 | 04 批次管理、05 批量测试、06 测试历史、07 包装管理 | 批次是测试、历史、包装的上游上下文 |
| 批量动作、人工判定、保存历史 | 05 批量测试、06 测试历史、03 仓库管理 | 人工判定与历史保存会反向影响仓库状态 |
| 检验台上传、绑定、历史快照删除 | 06 测试历史、04 批次管理、05 批量测试 | 历史与检验台变更要关注绑定和删除边界 |
| 包装扫码、装箱、下一箱、打印 | 07 包装管理、03 仓库管理、05 批量测试 | 包装强依赖仓库设备状态与测试通过约束 |

### 4.3 AI 执行建议

- 用户只说“先快速回归一下”时，默认按模块最小回归集执行
- 用户说“这个改动会影响哪些测试”时，先按“变更类型”选模块，再进入各模块 `测试用例.md`
- 遇到字段、状态、权限、删除守卫、多租户相关改动时，不允许只跑单模块 smoke，至少补跑上游和下游一个关联模块

## 5. 回归触发规则

以下任一改动都要回归本页：

- 登录与鉴权
- 多租户逻辑
- 仓库状态字段
- 导入/导出
- 设备上报、回执、超时链路
