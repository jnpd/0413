# 发布流程

## 文档信息

| 项 | 内容 |
|---|---|
| 最后更新 | 2026-04-13 |
| 当前状态 | 项目级最小流程，真实部署平台待补 |

## 1. 发布前检查

- proposal / spec / design / contracts / test-cases 是否齐全
- 高风险规则是否已更新知识页
- smoke 是否已跑通
- baseline 是否已补齐
- 需要执行的 SQL 是否已确认
- 回滚条件是否已写清楚

## 2. 最小发布步骤

1. 确认本次变更范围与非范围
2. 执行相关 SQL 升级脚本
3. 启动或更新后端服务
4. 验证登录与关键业务接口
5. 记录发布结果与异常

## 3. 当前建议 smoke

- `POST /blade-auth/oauth/token`
- `GET /blade-wms/warehouse/stat`
- `GET /blade-wms/warehouse/page`

## 4. 回滚条件

- 登录链路不可用
- 仓库核心接口不可用
- 多租户/删除守卫等高风险规则回归
- SQL 升级后出现不可接受的数据破坏风险

## 5. 回滚步骤

1. 停止继续扩大发布范围
2. 回退本次代码变更
3. 按预先确认的数据库策略处理回滚
4. 重新执行最小 smoke
5. 把事故回灌到 `docs/07-incidents/缺陷案例索引.md`

## 6. 当前缺口

- 部署平台与审批流待补
- 自动化发布命令待补
- SQL 回滚脚本待补
