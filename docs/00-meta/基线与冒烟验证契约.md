# 基线与冒烟验证契约

## 1. 目的

统一 baseline 检查和 smoke 验证的顺序与标准。

## 2. baseline 回答什么

baseline 回答的是：

- 这个模块像不像一个能交付的模块

固定检查：

- 页面结构是否完整
- 状态是否完整
- 权限是否完整
- 提交态是否完整
- 错误反馈是否完整
- 契约是否一致

## 3. smoke 回答什么

smoke 回答的是：

- 主链到底通没通

固定检查：

- 能不能进
- 能不能查
- 能不能填
- 能不能提
- 能不能成
- 能不能看到结果

## 4. 固定顺序

```text
先主链 smoke
→ 再边界 smoke
→ 再基线补漏
→ 再验收
```

## 5. AI 使用要求

- 主链没通前，不允许 AI 先做大量边界优化
- baseline 和 smoke 没有结果时，不允许 AI 宣称完成
