# 通用契约索引

## 1. 说明

当前仓库不再单独保留顶层通用契约目录。

通用契约已按主题拆分到现有 `00-07` 目录下，原则如下：

- 项目级文档优先于通用契约
- 只保留现有顶层目录结构，不新增新的数字目录
- 需要引用通用契约时，按下表进入对应分区

通用契约的分类原则见：`docs/02-rules/README.md`

## 2. 优先级

```text
change/contracts 或后续补齐的规格真源
> docs/02-rules/ 中的项目级强规则
> 分散落盘的通用契约
> 外部 starter 模板
```

## 3. 分布位置

### 00-meta

- [交付护栏契约.md](./交付护栏契约.md)
- [缺口台账.md](./缺口台账.md)
- [change-writing-anti-drift-guide.md](./change-writing-anti-drift-guide.md)

适用判断：

- 你在裁决 change 写法、交付护栏、知识缺口、历史跑偏案例与规范入口

## 4. 使用方式

- 写项目规则时，先读项目级正文，再补读对应契约。
- 需要做 change 编写、交付边界、缺口盘点时，优先从 `00-meta` 进入。
- 如果某条规则只服务于一个模块、一个状态机、一个字段族或一个 feature，不要放进通用契约。
