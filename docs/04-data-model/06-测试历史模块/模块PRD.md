# 测试历史模块 PRD

## 1. 模块定位

当前测试历史模块对应：

- 前端组件：`frontEnd/src/views/product/batchTestingHistory/history.tsx`
- 后端接口前缀：`/blade-wms/testing-history`

该模块当前覆盖两部分：

- 批量测试历史列表与详情
- 检验台 Excel 数据管理与绑定

## 2. 当前页面结构

### 2.1 历史列表页

当前列表页支持：

- 搜索表号（1-14 位纯数字，少于 14 位自动补零）
- 搜索备货单号 / 测试批号 / 表具名称
- 分页
- 分组展开

当前列表字段：

- 备货单号
- 测试批号
- 备注
- 设备总数
- 合格
- 异常
- 待测
- 测试进度
- 操作

### 2.2 历史详情页

详情页显示：

- 批次编号
- 测试时间
- 操作员
- 合格率
- 明细表

明细字段：

- 测试任务号
- 表号(平台)
- IMEI
- 测试时间
- 合格状态
- 设定流量(L/h)
- 实际流量(L/h)
- 温度(℃)
- 密度(kg/L)
- 标准值(L)
- 相对误差(%)
- 是否合格

### 2.3 检验台区

当前页面右侧检验台区支持：

- 查看检验台记录列表
- 上传 Excel
- 新增空白检验台记录
- 删除检验台记录
- 查看某条检验台记录下的全部导入行

## 3. 当前页面能力

### 3.1 历史列表操作

当前每个批次或分组能看到的操作，实际包括：

- 查看详情
- 配置检验台
- 删除单条历史快照（仅删除当前 `snapshotId`）

当前页面里还有一些 UI 元素，但未真正接线：

- “导出详情”按钮：当前无导出逻辑
- 日历按钮：当前无日期筛选逻辑

### 3.2 检验台 Excel 上传

当前上传要求：

- 仅支持 `.xlsx`
- 模板列必须包含：
  - 表号
  - 设定流量(L/h)
  - 实际流量(L/h)
  - 温度(℃)
  - 密度(kg/L)
  - 标准值(L)
  - 相对误差(%)
  - 是否合格

当前后端强校验每个表号必须正好有 3 个流量点：

- `2500`
- `40`
- `25`

当前前端交互要求：

- 点击“确认上传”后，提交按钮必须进入 loading，按钮文案切换为“数据正在上传中...”
- 上传进行中，弹窗内必须显示“数据正在上传中，请勿关闭弹窗或刷新页面，上传完成后会自动刷新当前检验台记录。”
- 上传进行中禁止重复提交，也不允许关闭上传弹窗

### 3.3 配置检验台

当前前端“配置检验台”的真实行为不是批量自动配置接口，而是：

1. 先在左侧选择 1 个平台表号
2. 再在右侧选择 1 条检验台记录
3. 前端高亮该表号对应的 3 条流量点
4. 点击“匹配 3 条”
5. 最终调用 `/bench/bind`

因此当前页面实际是“单表号、三条流量点绑定”，不是老文档里描述的整批自动匹配。

当前前端交互要求：

- 弹窗标题区保留显式“关闭”按钮，避免用户滚动到底部才能退出。
- 自动配置执行中禁用关闭入口，防止用户误以为任务已取消。

### 3.4 当前前端未接入的后端能力

后端已有但当前前端未正式接入：

- `POST /bench/configure`
- `POST /bench/unbind`

前端 API 文件虽然已经封装了这些接口，但 `history.tsx` 当前没有调用它们。

### 3.5 检验台记录改名

前端有“编辑检验台记录”弹窗，但当前代码里明确提示：

- 后端暂无改名接口

因此当前编辑弹窗不能真正修改已存在记录名称，只能新建空白记录。

## 5. 当前代码规则

### 5.1 检验台记录删除

只有当检验台记录未被历史快照引用时，才允许删除。

### 5.2 历史快照删除

历史列表子行“删除”只删除当前这一条 `snapshotId`：

- 软删除 `tl_batch_snapshot`
- 软删除当前快照下的 `tr_batch_detail_history`
- 同步清理当前快照下的历史绑定关系
- 不删除全局检验台原始记录头和原始行

### 5.3 绑定规则

绑定时，后端会校验：

- 平台该流量点是否已绑定
- 检验台原始行是否属于当前选择的检验台记录头

### 5.4 历史明细

历史详情页读的是快照数据，不是实时批量测试页的当前态。

### 5.5 表号口径

- 历史列表搜索条件会把 1-14 位纯数字表号统一补零到 14 位后再查询
- 平台历史明细与检验台行回显时，纯数字表号统一按 14 位展示
