## 2. 核心表

### 2.1 历史快照

当前测试历史的上游快照表为：

- `tl_batch_snapshot`
- `tr_batch_detail_history`

其中：

- `tl_batch_snapshot` 保存批次级快照头
- `tr_batch_detail_history` 保存设备级快照明细

### 2.2 检验台记录头

`tl_test_data_sheet`

实体映射：

- `@TableName("tl_test_data_sheet")`

当前核心字段：

- `sheet_id`
- `snapshot_id`
- `batch_id`
- `bench_name`
- `file_name`
- `remark`
- `row_count`
- `upload_time`
- `tenant_id`
- `enterprise_id`
- `is_deleted`

说明：

- 当前上传 Excel 时，`snapshot_id`、`batch_id` 为空
- 新增空白记录时，可以带快照上下文，也可以为空

### 2.3 检验台原始行

`tl_test_data`

实体映射：

- `@TableName("tl_test_data")`

当前核心字段：

- `row_id`
- `sheet_id`
- `snapshot_id`
- `meter_no`
- `flow_point_seq`
- `flow_point_name`
- `set_flow`
- `actual_flow`
- `temperature`
- `density`
- `standard_value`
- `relative_error`
- `is_qualified`
- `tenant_id`
- `enterprise_id`
- `is_deleted`

### 2.4 历史绑定表

当前绑定关系由两张表共同表达：

- `tr_batch_detail_test`
- `tr_batch_history_test_rel`

对应实体：

- `TestingHistoryBenchBinding`
- `TestingHistoryBenchRelation`

当前作用：

- `tr_batch_detail_test` 保存历史明细与检验台行的绑定快照
- `tr_batch_history_test_rel` 保存历史明细到绑定记录的关系索引

## 3. 关键规则

### 3.1 Excel 模板规则

后端当前要求每个表号必须有且仅有这 3 个流量点：

- `2500` -> `Q1(大)` -> `flow_point_seq = 1`
- `40` -> `Q2(中)` -> `flow_point_seq = 2`
- `25` -> `Q3(小)` -> `flow_point_seq = 3`

缺失或重复都会直接拦截导入。

对于 Excel 或查询条件中的纯数字表号，当前口径为：

- 允许 1-14 位纯数字
- 入库/查询前统一左补零到 14 位
- 诸如 `6.0` 这类 Excel 数值会先规整为数字字符串，再补零

### 3.2 绑定规则

单次绑定时：

- 同一个历史明细 + 同一个流量点，不能重复绑定
- 同一条检验台原始行，不能被多个历史明细复用

### 3.3 删除规则

删除 `tl_test_data_sheet` 前，会检查：

- 是否仍有有效绑定记录引用该 `sheet_id`

有引用则不允许删除。

## 4. 关联关系

- `tr_batch_detail_history.snapshot_id -> tl_batch_snapshot.snapshot_id`
- `tl_test_data.sheet_id -> tl_test_data_sheet.sheet_id`
- `tr_batch_detail_test.snapshot_id -> tl_batch_snapshot.snapshot_id`
- `tr_batch_detail_test.history_id -> tr_batch_detail_history.history_id`
- `tr_batch_detail_test.row_id -> tl_test_data.row_id`
- `tr_batch_history_test_rel.history_id -> tr_batch_detail_history.history_id`
- `tr_batch_history_test_rel.test_id -> tr_batch_detail_test.binding_id`

## 5. 当前代码约束

### 5.1 企业隔离

检验台记录和原始行都带：

- `tenant_id`
- `enterprise_id`

非平台超管只能访问本企业的检验台记录。

### 5.2 历史详情回显

历史详情页中的检验台流量点数据，并不是现查 Excel 原始行，而是优先读取绑定快照：

- `flow_point_seq`
- `flow_point_name`
- `set_flow`
- `actual_flow`
- `temperature`
- `density`
- `standard_value`
- `relative_error`
- `is_qualified`

这保证历史页不会被后续 Excel 变更反向污染。

### 5.3 表号查询口径

- 历史快照明细和检验台原始行的表号匹配按 14 位补零口径处理
- 模糊搜索时会基于 `LPAD(meter_no, 14, '0')` 进行匹配，兼容旧数据与新数据

### 5.3 前端与后端当前接线差异

虽然数据库结构支持：

- 批量配置
- 手动解绑

但当前前端页面只实接：

- 新增记录
- 上传 Excel
- 查看原始行
- 绑定 3 条流量点
- 删除未引用记录

## 6. 与旧文档的修正点

旧文档把检验台能力写成“整批配置 + 完整 CRUD + 全量解绑流程”，当前代码并不是这样：

- 前端当前不是整批配置，而是单表号三条流量点绑定
- 前端没有显式解绑入口
- 前端没有真正的检验台重命名保存接口

因此本文件按当前数据库实际用途重写，不再沿用旧的功能叙述。
