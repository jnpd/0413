# 表具档案模块 PRD

## 1. 模块定位

当前表具档案模块只有一个实际页面：

- 前端组件：`frontEnd/src/views/product/archive/index.tsx`
- 后端接口前缀：`/blade-wms/meter-archive`

该模块负责维护“表具基础档案”，供仓库入库、包装选型等下游模块引用。

## 1.1 提交交互约束

表具档案模块中的新增、编辑、删除都必须使用整体界面阻断式 loading，提交中禁止继续触发其它写操作。

按钮级 loading 只用于补充反馈，不能替代覆盖当前页面与当前弹窗上下文的整体提交遮罩。

## 2. 当前页面能力

### 2.1 列表能力

当前页面支持：

- 关键字搜索
- 分页
- 列设置
- 排序
- 新增档案
- 编辑档案
- 删除档案
- 搜索词、排序或刷新条件变化时先回到第一页，并收敛重复列表请求

搜索范围：

- 表具名称
- 表具型号
- 表具口径
- 备注

支持排序的列：

- 表具名称
- 表具型号
- 表具口径
- 备注
- 创建时间

### 2.2 列表字段

当前列表字段为：

- 表具名称
- 表具型号
- 表具口径
- 备注
- 创建时间
- 操作

### 2.3 新增/编辑表单

当前表单字段：

- 表具名称
- 表具型号
- 表具口径
- 备注

字段规则：

- 表具名称必填
- 表具型号必填
- 表具口径必填
- 备注可空

## 3. 当前代码规则

### 3.1 唯一性

后端当前按租户校验：

- `meter_name` 唯一

当前没有看到按“名称 + 型号 + 口径”联合唯一的实现，旧文档中如果写了联合唯一，应以当前代码为准进行删除。

### 3.2 删除约束

删除前，后端会检查是否被下游引用：

- `tb_device`
- `tl_package_box`

只要有引用，就不允许删除。

### 3.3 详情接口

后端有 `detail` 接口，但当前页面主要使用分页和提交接口；没有独立详情页。

## 4. 当前接口

| 能力 | 接口 |
| --- | --- |
| 分页查询 | `GET /blade-wms/meter-archive/page` |
| 详情 | `GET /blade-wms/meter-archive/detail` |
| 下拉列表 | `GET /blade-wms/meter-archive/list` |
| 新增/编辑 | `POST /blade-wms/meter-archive/submit` |
| 删除 | `POST /blade-wms/meter-archive/remove` |

## 5. 与下游模块关系

- 仓库管理入库时必须选择表具档案
- 包装管理创建包装记录时需要选择表具档案
- 页面下拉显示名称来自 `meterName / meterModel / meterCaliber` 拼接后的 `displayName`

## 6. 当前未实现的内容

当前代码里没有这些能力：

- 批量导入表具档案
- 表具档案导出
- 独立详情页
- 状态筛选页签
