## 1. 代码依据

主要依据：

- `modules/wms/archive/pojo/entity/MeterArchive.java`

## 2. 核心表

### 2.1 `tb_meter`

实体映射：

- `@TableName("tb_meter")`

当前实体字段：

- `meter_id`
- `tenant_id`
- `meter_name`
- `meter_model`
- `meter_caliber`
- `remark`
- `create_user`
- `create_time`
- `update_user`
- `update_time`
- `status`
- `is_deleted`

## 3. 关键字段说明

### 3.1 业务字段

- `meter_name`：页面主显示字段，按租户唯一
- `meter_model`：型号
- `meter_caliber`：口径
- `remark`：备注

### 3.2 审计字段

- `tenant_id`
- `create_user`
- `create_time`
- `update_user`
- `update_time`
- `status`
- `is_deleted`

## 4. 关联关系

当前代码中的下游引用主要有两处：

- `tb_device.meter_id -> tb_meter.meter_id`
- `tl_package_box.meter_id -> tb_meter.meter_id`

这也是删除校验的直接依据。

## 5. 当前代码约束

### 5.1 唯一性

后端当前只校验：

- 同租户下 `meter_name` 不能重复

### 5.2 删除

删除时不是物理删，而是：

- 把 `is_deleted` 置为 1

但执行前会检查：

- 是否已被仓库设备引用
- 是否已被包装箱引用

存在引用则抛错，不允许删除。

### 5.3 排序

分页接口支持以下排序映射：

- `name -> meter_name`
- `model -> meter_model`
- `caliber -> meter_caliber`
- `remark -> remark`
- 默认 -> `create_time`

## 6. 与旧文档的修正点

当前代码未体现：

- 复杂 ER 图
- 多表拆分
- 联合唯一约束
- 独立状态机

因此本模块数据库设计以单表 `tb_meter` 为准。
