# 批量测试模块 PRD

## 1. 模块定位

当前批量测试模块对应：

- 前端组件：`frontEnd/src/views/product/BatchTesting/index.tsx`
- 后端接口前缀：`/blade-wms/batch-testing`

当前页面必须先选定一个测试批次，才能继续操作。

## 2. 当前页面结构

### 2.1 顶部批次上下文

当前页面顶部会显示：

- 当前批次下拉
- 新增设备按钮
- 异常判定参数按钮
- 当前备货单号
- 当前批次创建时间
- 统计卡片

统计卡片只有 4 类：

- 总数
- 合格
- 不合格
- 测试中

### 2.2 展开筛选区

当前展开筛选区支持：

- 按备货单号筛选
- 按关键字即时筛选
- 判定状态筛选：全部 / 测试中 / 合格 / 不合格

关键字搜索范围在前端文案中明确包括：

- 表号
- IMEI
- 流量
- 无磁信号
- 温压
- CSQ / RSRP / RSRQ
- 主副 IP
- 通讯协议

纯数字表号在查询与回显时按 14 位口径处理：

- 输入 1-14 位纯数字时，查询前会自动补零到 14 位
- 列表回显纯数字表号时统一展示为 14 位

## 3. 当前列表字段

当前表格字段为：

- IMEI
- 表具名称
- 水表表号
- 上报时间
- 阀门状态
- 电池电压
- 正向累积流量
- 反向累积流量
- 备货单号
- 合格状态
- 无磁信号
- 服务器 IP / 端口
- 上报周期
- 环境温度
- 压力
- CSQ / RSRP / RSRQ
- 上报成功/上报次数
- 通讯协议
- 写表时间
- 操作人员
- 操作

单行操作当前只有：

- 标记合格
- 标记不合格
- 移除当前批次

## 4. 当前页面操作

### 4.1 新增设备

当前新增设备弹窗标题为“扫描设备并绑定备货单/批次”。

真实规则：

- 当前只接受 15 位 `IMEI`
- 当前只支持通过 `IMEI` 加入批次
- 扫码后回车会立即校验并提交
- 同设备只允许绑定唯一备货单号
- 同设备可以加入多个批次

虽然代码里保留了更多标识类型枚举，但当前前后端实际校验都只放行 `IMEI`。

### 4.2 批量动作

当前页面展示的动作按钮为：

- 数据上报
- 开阀
- 关阀
- 写表底
- 写表号
- 读取上报周期
- 设置上报周期
- 写 IP 地址
- 程序升级
- 查看原始数据
- 写表时间
- 批量合格
- 保存历史

其中当前真实状态如下：

| 动作 | 当前状态 |
| --- | --- |
| 数据上报 | 已接入 |
| 开阀 | 已接入 |
| 关阀 | 已接入 |
| 写表底 | 已接入 |
| 写表号 | 已接入 |
| 读取上报周期 | 已接入 |
| 设置上报周期 | 已接入 |

补充说明：

- 后端命令规则中已支持 `READ_IP`，但当前页面没有“读取 IP 地址”按钮
- 行内编辑接口 `/device/update` 存在，但当前页面没有显式编辑按钮
- 开阀 / 关阀要求所选设备都具备表号和通讯协议，否则前端直接禁用并提示原因

### 4.3 任务详情与原始数据

当前页面支持：

- 查看批量动作任务详情
- 查看单设备原始数据

任务详情展示：

- 任务号
- 动作类型
- 任务状态
- 成功/失败/超时汇总
- 逐台执行结果
- request raw / response raw / 返回 JSON

原始数据弹窗展示：

- 数据源
- 报文时间
- 方向
- 报文内容

### 4.4 保存历史

当前“保存历史”会调用后端快照接口，把当前批次的实时结果固化为历史快照。

页面传的备注固定是：

- `页面保存历史`

保存成功后会提示：

- 批次号
- 当前设备总数

## 5. 异常判定参数

### 5.1 当前页面可配项

当前页面只允许配置 3 个“异常标红阈值”：

- 电池电压下限
- CSQ 下限
- 无磁信号下限

### 5.2 当前代码口径

这些参数只用于：

- 列表标红

它们不等同于最终“合格/不合格”判定。

默认值来自后端：

- 电池电压：`3.6`
- CSQ：`13`
- 无磁信号：`130`

### 5.3 当前未接入项

条件图片存档相关 UI 代码仍在注释中，当前页面不可用。

## 7. 当前代码规则

### 7.1 新增设备

- 当前只允许扫描 15 位 IMEI
- 如果设备已绑定其他备货单号，不允许重新绑定
- 如果设备已在当前批次，不允许重复加入

### 7.2 更新设备

当前后端只允许修改 3 个字段：

- `meterNo`
- `presetBaseValue`
- `remark`

其中 `meterNo` 当前规则为：

- 允许 1-14 位纯数字
- 少于 14 位自动补零
- 保存与回显都按补零后的 14 位口径处理

### 7.3 移除设备

如果设备已经进入包装流程，则不允许从当前批次移除。

### 7.4 写表号

- 前端写表号时只接受 1-14 位纯数字，少于 14 位自动补零
- 后端会按补零后的 `meterNo` 检查当前批次及仓库设备唯一性
- 成功后会同步回写 `tb_device.meter_no`

### 7.5 人工判定同步

- 批量合格 / 不合格除了更新批次当前态，还会同步回写 `tb_device.test_status`

## 8. 当前未接入或仅后端预留的能力

- `READ_IP`：后端已支持命令码，前端未暴露按钮
- `UPGRADE_PROGRAM`：前端按钮存在，但当前不走后端闭环
- 行级编辑：接口存在，但当前页面没有正式编辑入口
