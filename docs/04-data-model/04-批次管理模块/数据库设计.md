## 3. 统计与删除约束依赖

批次管理页显示的统计和删除能力并不只来自本模块两张主表，还依赖下游业务表：

| 依赖表 | 用途 |
| --- | --- |
| `tr_batch_detail_device` | 统计设备总数 / 合格 / 异常 / 待测 |
| `tl_batch_snapshot` | 判断批次是否已生成历史快照 |
| `tr_batch_detail_test` | 判断批次是否已绑定检验台数据 |
| `tl_package` | 判断批次是否已产生包装记录 |

说明：

- 这些表不是批次管理模块主表，但它们直接影响列表统计和删除按钮是否可用。

## 4. 关键规则

### 4.1 编码规则

- `prep_no` 格式：`2603-001`
- `batch_no` 格式：`2603-001-P001`

后端按当前 `prep_id` 下已有批次号自动计算下一序号。

### 4.2 备货单号重命名

重命名备货单号时：

- 更新 `tl_stock_prep.prep_no`
- 同步更新该组下所有 `tl_batch.batch_no`

如果目标批次号已存在，后端直接拦截。

### 4.3 删除

删除 `tl_stock_prep`：

- 仅当该 `prep_id` 下不存在有效 `tl_batch` 记录

删除 `tl_batch`：

- 仅当无历史快照
- 无检验台绑定
- 无包装记录

## 5. 关联关系

- `tl_batch.prep_id -> tl_stock_prep.prep_id`
- `tb_device.prep_id -> tl_stock_prep.prep_id`
- `tr_batch_detail_device.batch_id -> tl_batch.batch_id`
- `tl_batch_snapshot.batch_id -> tl_batch.batch_id`
- `tl_package.prep_id -> tl_stock_prep.prep_id`

## 6. 当前代码约束

### 6.1 企业隔离

`tl_stock_prep` 和 `tl_batch` 都带：

- `tenant_id`
- `enterprise_id`

列表查询和下拉查询都会按当前企业范围过滤。

### 6.2 软删除

两张主表都不是物理删除，而是依赖：

- `is_deleted`
- `status`

做有效记录判断。

### 6.3 分页结果结构

当前分页接口返回的 `records` 不是单纯 `tl_stock_prep` 或 `tl_batch` 原始行，而是：

- 先返回 1 条备货单号分组行
- 再返回其下若干条批次明细行

前端再按 `batchNo === ''` / `batchNo !== ''` 区分分组行和明细行。
