# 审计追踪契约

## 1. 目的

统一操作审计和请求追踪，保证出现问题后能回查。

## 2. 固定字段

- `createdBy`
- `createdAt`
- `updatedBy`
- `updatedAt`
- `traceId`

按项目需要可补：

- `tenantId`
- `operatorId`
- `operatorName`

## 3. 固定约束

- 写操作必须可追踪到操作人
- 请求必须可追踪到 `traceId`
- 列表和详情中需要展示时，优先统一字段名
- 审计字段默认由后端维护，不允许前端伪造

## 4. 日志定位要求

- 错误响应里必须带 `traceId`
- 后端日志必须能通过 `traceId` 反查
- 关键写操作建议记操作日志

## 5. AI 使用要求

- 涉及写操作、审批流、批量任务时，默认要求补审计和追踪字段
