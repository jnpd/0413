# 统一响应契约

## 1. 目的

统一后端响应壳，避免前端、后端、AI 各写一套返回格式。

## 2. 固定字段

- `code`: 结果码。`0` 表示成功，非 `0` 表示失败
- `message`: 面向前端展示或排障的结果描述
- `data`: 业务数据。成功时返回对象、数组或 `null`
- `traceId`: 请求追踪标识，用于日志定位

## 3. 固定约束

- 不要同时返回多套成功标识，例如同时出现 `success`、`code`、`status`
- 前端只以 `code` 判断成功失败
- `message` 不能为空
- 即使失败，也要返回统一壳
- `traceId` 必须可追踪到后端日志

## 4. 标准示例

成功：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1001
  },
  "traceId": "7f1f7b83b7e94c31"
}
```

失败：

```json
{
  "code": 40001,
  "message": "用户名已存在",
  "data": null,
  "traceId": "7f1f7b83b7e94c31"
}
```

## 5. AI 使用要求

- 新项目默认直接复用这套响应壳
- 如果项目要改字段名，必须在项目级契约里显式声明
- 没有项目级覆盖时，不允许 AI 自行发明新的返回结构
