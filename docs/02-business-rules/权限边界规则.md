# 权限边界规则

## 文档信息

| 项 | 内容 |
|---|---|
| 最后更新 | 2026-04-13 |
| 风险级别 | 高 |
| 当前锚点 | `application.yml`、`AGENTS.md`、后端系统模块代码 |

## 1. 目标

明确页面权限、按钮权限、接口权限、数据权限的边界，防止“看得见”和“做得到”混成一层。

## 2. 固定分层

- 页面权限：是否允许进入模块
- 按钮权限：是否允许触发某个操作
- 接口权限：是否允许真正执行请求
- 数据权限：即使接口可访问，也只能处理授权范围内的数据

## 3. 当前已验证事实

- `strict-token = true`
- `strict-header = true`
- 系统模块中已存在菜单、角色、数据权限相关实体与服务

## 4. 项目级规则

### 4.1 后端为最终裁判

- 前端权限控制只做体验优化，不承担最终安全责任。
- 只要涉及删除、出库、导入、跨租户访问，后端都必须再次校验。

### 4.2 无权限反馈要稳定

- 无页面权限：禁止进入页面或统一提示
- 无操作权限：按钮隐藏/禁用与后端拒绝语义要一致
- 无数据权限：必须明确为越权，不得伪装成“数据不存在”

### 4.3 高风险动作默认更严

- 删除
- 出库
- 导入
- 角色/菜单权限配置

上述动作不能只靠页面隐藏按钮，需要接口与数据范围双重校验。

## 5. 当前缺口

- 当前仓库模块代码里尚未看到显式按钮级权限注解或数据范围拦截说明。
- 真实菜单 code、角色授权表与前端动态菜单入口仍待补齐。

## 6. 测试要求

- 页面进入权限
- 按钮显示/不可用行为
- 接口越权拒绝
- 数据越权拒绝
