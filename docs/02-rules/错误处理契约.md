# 错误处理契约

## 1. 目的

统一错误类型和前端展示方式，避免错误反馈混乱。

## 2. 固定错误分类

- 参数错误
- 业务错误
- 权限错误
- 系统错误

## 3. 固定处理原则

- 参数错误：提示用户修正输入
- 业务错误：给出可读业务原因
- 权限错误：提示无权限，并阻断后续操作
- 系统错误：提示通用失败信息，并提供 `traceId`

## 4. 固定展示约定

- 表单内校验错误：优先字段级提示
- 提交失败：优先消息提示，必要时保留字段级错误
- 页面级请求失败：优先错误态或全局错误提示
- 危险操作失败：必须给明确失败原因

## 5. 不允许的做法

- 后端直接把技术异常栈暴露给前端
- 前端把所有错误都显示成“操作失败”
- 参数错误和业务错误不区分

## 6. AI 使用要求

- 只要设计接口，就默认要求设计错误语义
- 只要设计页面，就默认要求设计错误展示方式
