## 1. 项目级覆盖说明

本项目的接口口径与通用契约有两处已知差异：

- 响应壳采用 BladeX `R` 风格，不是《统一响应契约》里的 `code=0/message/data/traceId`
- 分页参数与返回采用 `current/size/records/total`，不是 `pageNum/pageSize`

因此：

- 通用契约用于统一思路
- 本页用于冻结当前项目真实接口口径

## 2. 鉴权约定

### 2.1 登录接口

- `POST /blade-auth/oauth/token`
- 默认租户参数：`tenant_id=000000`
- 密码需按 `04 + sm2.doEncrypt(password, publicKey, 0)` 加密

### 2.2 业务请求头

- `Authorization: Basic c2FiZXIzOnNhYmVyM19zZWNyZXQ=`
- `Blade-Auth: bearer <access_token>`
- `Blade-Requested-With: BladeHttpRequest`

### 2.3 大数字处理

- `blade.jackson.big-num-to-string = true`
- 前后端都应按 Long 可能被序列化为字符串处理

## 3. 仓库接口清单

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/blade-wms/warehouse/stat` | 统计卡片 |
| GET | `/blade-wms/warehouse/page` | 分页查询 |
| GET | `/blade-wms/warehouse/detail` | 单条详情 |
| POST | `/blade-wms/warehouse/submit` | 新增/编辑 |
| POST | `/blade-wms/warehouse/remove` | 删除 |
| POST | `/blade-wms/warehouse/import` | 导入占位接口 |
| POST | `/blade-wms/warehouse/outbound` | 批量出库 |

## 4. 仓库分页口径

### 4.1 入参

- `current`
- `size`
- `keyword`
- `stockStatus`
- `testStatus`

### 4.2 出参核心字段

- `records`
- `total`

## 5. 仓库提交字段

当前已确认的提交字段：

- `deviceId`
- `meterId`
- `prepId`
- `imei`
- `protocolType`
- `carrier`
- `vendorCode`
- `remark`

## 6. 当前已确认错误语义

| 场景 | 错误文案 |
|---|---|
| `meterId` 缺失 | `表具档案不能为空` |
| `imei` 缺失 | `IMEI 不能为空` |
| `imei` 格式错误 | `IMEI 必须为15位数字` |
| `protocolType` 缺失 | `通讯协议不能为空` |
| `carrier` 非法 | `运营商不存在` |

## 7. 高风险约束

- 多租户范围以后端为准
- 删除守卫、导入规则、重复出库阻断虽未完全实现，但在接口层面必须保留稳定错误语义
- 前端错误提示优先透传后端可读文案
