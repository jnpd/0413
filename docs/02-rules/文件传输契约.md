# 文件传输契约

## 1. 目的

统一上传、下载、导入、导出类能力的交互和接口约定。

## 2. 固定场景

- 单文件上传
- 多文件上传
- 模板下载
- 数据导出
- 批量导入

## 3. 固定契约

- 上传前明确文件类型限制
- 上传前明确文件大小限制
- 上传失败必须返回可读原因
- 导入失败必须给错误报告或错误明细
- 导出长耗时任务必须说明同步还是异步

## 4. 页面行为约定

- 上传前展示支持格式和大小
- 上传中展示进度或提交中态
- 上传成功给明确反馈
- 上传失败保留错误原因
- 导入失败支持定位哪一行或哪一项出错

## 5. 接口约定

- 上传接口明确字段名，例如 `file`
- 导出接口明确返回文件流还是异步任务 ID
- 模板下载接口固定返回文件流
- 失败时仍返回统一响应壳或统一错误结构

## 6. AI 使用要求

- 只要出现导入导出或文件上传下载，默认引用这份契约
- 没有项目级说明时，不允许 AI 省略格式限制、大小限制和失败反馈
