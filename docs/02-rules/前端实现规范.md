## 三、全局布局原则

> 状态：当前适用

| 项目 | 规范 |
|------|------|
| 适配范围 | 桌面端优先，不考虑移动端适配 |
| 页面高度 | 页面容器占满屏幕可用高度，主内容区随视口高度变化自动伸缩 |
| 页面级外边距 | 管理类列表页四周统一收紧为 `16px` |
| 首屏目标 | 顶部操作区、主体内容区、分页或底部操作区默认需在一屏内可见，避免页面级纵向滚动 |
| 滚动策略 | 页面整体不滚动，列表区、弹窗内容区、详情页右侧内容区承担内部滚动 |
| 表格策略 | 表头固定、表体滚动；横向溢出时保留关键操作列可见 |
| 最小可用性 | 列表区至少保证 5 行数据可见 |

**实现要求**
- 页面主框架应优先采用 `flex` 布局实现高度自适应。
- 页面级容器应避免通过写死整体高度实现布局。
- 视口尺寸变化时，布局应自动响应，无需用户刷新页面。
- 当可用高度不足时，只允许内容区滚动，不应把顶部操作和底部关键操作挤出首屏。

## 四、主内容区布局规范

> 状态：当前适用

### 4.1 页面标题区

| 属性 | 规范 |
|------|------|
| 背景色 | `#ffffff` |
| 内边距 | 上下 20px，左右 24px |
| 标题字号 | 20px |
| 标题字重 | 600 (Semi-bold) |
| 标题颜色 | `#111827` |
| 描述字号 | 14px |
| 描述颜色 | `#6b7280` |
| 图标尺寸 | 24px × 24px |

**布局要求**
- 标题左侧可配置图标，与标题文字垂直居中对齐
- 描述文字位于标题下方，间距 8px
- 右侧可放置全局操作按钮（如"新增项目"）

### 4.2 操作区布局

| 属性 | 规范 |
|------|------|
| 背景色 | `#ffffff` |
| 内边距 | 上下 16px，左右 24px |
| 搜索框宽度 | 桌面端约 `23vw`，建议使用 `clamp(280px, 23vw, 360px)`，禁止拉伸占满整行 |
| 按钮间距 | 12px |
| 按钮高度 | 36px |

**布局要求**
- 采用 `justify-between` 布局，左右两侧分别对齐
- 左侧：搜索框 + 筛选下拉框 + `搜索`按钮 + `重置`按钮（使用 `flex-wrap` 支持换行）
- 右侧：操作按钮组（刷新 + 新增 + 返回等）
- 搜索框与按钮垂直居中对齐
- 搜索框应保持紧凑宽度，不得在桌面端占据整个操作区
- 所有元素在同一行显示，小屏幕下允许换行
- 管理类页面顶部操作区与上方容器的间距建议压缩到 `8px`
- 搜索区、表格区、分页区之间的纵向间距建议统一为 `8px`

**搜索区交互规范**
- 所有列表型页面的搜索区必须提供`搜索`按钮和`重置`按钮，不允许只保留输入框或筛选下拉框。
- 输入框内容和筛选项修改后先停留在草稿态，点击`搜索`按钮或在搜索框内按回车后才正式应用到列表。
- 点击`重置`按钮必须清空关键词、恢复筛选默认值、回到第 1 页，并立即刷新当前列表结果。
- 点击`刷新`按钮时必须保留当前已生效的搜索条件、筛选条件和分页参数。
- `搜索`按钮建议使用主按钮样式，`重置`按钮建议使用次要按钮样式。

**页面标题规范**
- 列表页面不显示大标题（如"项目管理"、"摄像头管理"）
- 通过左侧导航菜单的选中状态即可明确当前页面
- 操作区直接从搜索和筛选开始，节省垂直空间

**返回按钮规范**
- 子页面（如柜体管理）的返回按钮放在操作区右侧，与其他操作按钮对齐
- 返回按钮使用次要按钮样式（`secondaryButtonClass`）
- 返回按钮文字简洁（"返回"），配合左箭头图标
- 按钮顺序：刷新 → 新增 → 返回

**提示信息规范**
- 不在页面中显示技术实现细节的提示信息框
- 避免使用蓝色/黄色背景的提示框占用垂直空间
- 技术说明应放在开发文档中，而非用户界面
- 成功、失败、信息类轻提示统一使用顶部浮层 `Toast`，禁止以内联提示条插入搜索区、表格区或分页区上方
- 接口异常反馈文案必须优先使用后端返回的业务文案，取值顺序统一为 `msg` -> `error_description` -> 前端兜底文案；禁止把后端明确错误覆盖成通用“保存失败”“操作失败”
- 新增、编辑、删除、授权、提交审核等写操作接口失败时，必须触发错误 `Toast`；即使页面本身保留局部错误态，也不能缺少顶部全局反馈
- `Toast` 必须脱离文档流渲染，建议使用 `fixed + portal`，不得因为提示出现导致页面高度变化或列表抖动
- `Toast` 顶边必须贴齐视口顶部显示，不得下压到搜索区或表格上方中部位置，避免遮挡核心操作区域
- `Toast` 采用轻量横幅样式，不使用投影阴影；建议使用浅色背景 + 底部分隔线表达层级
- `Toast` 默认停留 `3s` 后自动消失，并支持手动关闭
- 同一时刻仅展示一条操作反馈类 `Toast`；新提示出现时覆盖旧提示，避免页面堆叠多个反馈框
- 阻断式异常或需要用户决策的场景不使用 `Toast`，应改用确认弹窗、错误占位或明确的页面态

**写操作提交态规范**
- 所有新增、编辑、删除、导入、授权、批量动作、保存历史、下一箱等写操作，必须显示整体界面阻断式 `loading`；只给按钮加 `loading` 不算达标。
- 整体界面 `loading` 必须覆盖当前页面与当前弹窗上下文，防止用户在提交未完成时继续点击其它写操作入口。
- 提交中必须同时禁止重复提交、关闭弹窗、键盘关闭、遮罩关闭；允许保留现有按钮级 `loading`、表单禁用和提示文案作为补充，但不能替代整体遮罩。
- 整体界面 `loading` 文案必须使用当前动作语义，例如“正在保存仓库设备，请勿操作”“正在执行批量操作，请勿操作”，禁止统一写成无差别“加载中”。
- 搜索、刷新、分页、只读详情加载、列表首次加载不使用整体界面阻断式 `loading`，避免误伤正常浏览操作。
- 新增页面默认复用共享页面容器提供的提交遮罩能力，不允许每个页面各自实现一套散落的全屏 `loading` 逻辑。

## 五、项目列表页布局规范

> 状态：当前适用

### 5.1 区域划分

| 区域 | 规范 |
|------|------|
| 页面容器 | 占满视口高度，减去顶部导航和面包屑高度 |
| 页面标题区 | 固定高度，根据内容自适应（建议 80-100px） |
| 操作区域 | 固定 68px |
| 列表区域 | 占满剩余空间，建议使用 `flex: 1` 实现 |
| 分页控件 | 固定 56px，位于页面底部 |

### 5.2 一屏展示要求

- 列表页默认状态下，搜索区、表格区、分页区需同时处于首屏可见范围内。
- 页面本身不出现纵向滚动条，表格内容超出时由表体区域滚动。
- 顶部操作区建议采用左侧搜索筛选、右侧操作按钮的对齐方式。
- 列表区域高度可通过弹性布局实现；如需计算高度，可使用 `calc(100vh - 顶部区域 - 分页区域 - 外边距)`。
- 所有列表型页面都必须保留分页区；即使当前数据量不足一页，也不能移除分页控件。
- 分页控件默认展示：首页、上一页、页码（最多 5 个）、下一页、末页，并显示当前页与总条数。
- 分页控件必须支持切换每页显示条数，可选项为：8、10、20、30、50、100 条。

### 5.2.1 带顶部统计卡片的列表页

- 带统计卡片的列表页仍然必须满足“一屏展示”：工具栏、统计卡片、表格区、分页区在桌面端默认状态下同时可见。
- 页面根容器必须使用 `h-full + min-h-0 + overflow-hidden` 一类弹性布局，禁止让整页内容随统计卡片增高而产生页面级纵向滚动。
- 统计卡片区必须视为顶部固定区，使用 `shrink-0` 保持自身高度稳定；表格区使用 `flex: 1` 吃满剩余高度，分页区固定贴底。
- 统计卡片建议控制在单卡 `72px ~ 88px` 高度区间，内边距建议 `12px`，图标容器建议 `40px`，数值字号建议不超过 `24px`。
- 统计卡片之间、统计区与表格区之间的垂直间距统一为 `8px`，避免大留白挤压表格可视行数。
- 当统计卡片数量不超过 4 张时，桌面端优先单行展示；最小验收分辨率 `1366 x 768` 下不得因为统计区过高而把分页区挤出首屏。
- 工具栏中的搜索输入、筛选下拉、刷新/新增等按钮应统一使用紧凑高度，建议 `32px`，不得混用表单区的大尺寸控件样式。

### 5.3 分页控件规范

| 属性 | 规范 |
|------|------|
| 背景色 | `#ffffff` |
| 边框 | 1px `#e5e7eb` |
| 圆角 | 16px |
| 内边距 | 上下 12px，左右 16px |
| 最小高度 | 56px |
| 布局方式 | `flex justify-between`，小屏幕下垂直排列 |

**左侧信息区**
- 显示内容：`显示 1-8 条，共 25 条` + `第 1/4 页` + 每页条数选择器
- 字号：14px
- 颜色：`#6b7280`（辅助文字）
- 每页条数选择器：
  - 样式：下拉框，高度 28px
  - 选项：10、20、30、50、100
  - 默认值：8
  - 文字：`每页 [下拉框] 条`

**右侧按钮区**
- 按钮类型：首页、上一页、页码按钮（最多 5 个）、下一页、末页
- 按钮尺寸：高度 32px，最小宽度 32px
- 按钮间距：8px
- 按钮样式：
  - 默认：白色背景，灰色边框 `#d1d5db`
  - 当前页：蓝色背景 `#eff6ff`，蓝色边框 `#bfdbfe`，蓝色文字 `#2563eb`
  - 禁用：透明度 50%，禁用鼠标样式
- 页码显示逻辑：
  - 总页数 ≤ 5：显示全部页码
  - 总页数 > 5：显示当前页前后各 2 页，其余用省略号表示

**交互要求**
- 切换每页条数时，自动跳转到第 1 页
- 页码变化时，表格内容自动刷新
- 禁用状态的按钮不响应点击事件

### 5.4 表格滚动规范

| 元素 | 行为 |
|------|------|
| 表头 | 固定在顶部，不参与滚动 |
| 表体 | 垂直滚动 |
| 横向滚动 | 内容区横向滚动时，操作列保持可见 |

## 六、表格样式与交互规范

### 6.1 表格基础样式

| 属性 | 规范 |

| 背景色 | `#ffffff` |
| 边框颜色 | `#e5e7eb` |
| 表头背景 | `#f9fafb` |

### 6.1.1 表格响应式规范

**文本溢出处理**
- 表格单元格内容应保持单行显示，避免换行导致行高不一致
- 超出单元格宽度的文本使用省略号（...）截断
- 鼠标悬停时通过原生 `title` 属性显示完整内容
- 实现方式：使用 CSS `truncate` 类（`overflow: hidden; text-overflow: ellipsis; white-space: nowrap`）

**响应式间距**
- 表格单元格内边距应根据屏幕尺寸自适应调整
- 小屏幕（< 640px）：`px-2 py-3`（左右 8px，上下 12px）
- 中等屏幕（≥ 640px）：`sm:px-3`（左右 12px）
- 大屏幕（≥ 1024px）：`lg:px-4`（左右 16px）
- 实现方式：使用 Tailwind CSS 响应式工具类

**单元格宽度控制**
- 为需要截断的单元格设置最大宽度（如 `max-w-[200px]`）
- 不同屏幕尺寸可设置不同的最大宽度（如 `max-w-[150px] sm:max-w-[200px]`）
- 确保所有单元格添加 `whitespace-nowrap` 防止内容换行

### 6.2 状态标签规范

| 状态 | 背景色 | 文字颜色 | 说明 |
|------|--------|---------|------|
| 运行 | `#d1fae5` | `#065f46` | 绿色系，表示正常运行 |
| 调试 | `#fef3c7` | `#92400e` | 黄色系，表示调试中 |
| 离线 | `#f3f4f6` | `#6b7280` | 灰色系，表示离线状态 |
| 故障 | `#fee2e2` | `#991b1b` | 红色系，表示故障状态 |

**样式要求**
- 标签高度：24px
- 内边距：左右 12px
- 圆角：12px (完全圆角)
- 字号：12px
- 字重：500 (Medium)

### 6.3 操作按钮规范

**标准操作按钮**

| 类型 | 样式 | 颜色 | 使用场景 |
|------|------|------|---------|
| 详情 | 文字按钮 | `#2563eb` | 查看详情、跳转 |
| 编辑 | 文字按钮 | `#2563eb` | 编辑操作 |
| 删除 | 文字按钮 | `#dc2626` | 删除操作 |

**交互要求**
- 按钮间距：16px
- 悬停时显示下划线
- 点击后有 0.2s 的颜色过渡效果

**紧凑型操作按钮（用于表格操作列）**

为了在小屏幕下减少行高、提高数据可见性，表格操作列应使用紧凑型按钮：

| 属性 | 规范 |
|------|------|
| 按钮类型 | 纯文字按钮（不使用图标） |
| 内边距 | `px-2 py-1`（左右 8px，上下 4px） |
| 字号 | `text-xs`（12px） |
| 字重 | 500 (Medium) |
| 圆角 | `rounded`（4px） |
| 按钮间距 | `gap-1`（4px） |
| 容器布局 | `flex items-center`（不使用 `flex-wrap`，避免换行） |

**样式类定义**
```typescript
// 基础紧凑按钮
compactButtonClass = 'inline-flex items-center justify-center whitespace-nowrap rounded px-2 py-1 text-xs font-medium transition-colors hover:bg-slate-100'

// 次要操作（蓝色）
compactSecondaryButtonClass = 'text-blue-600 hover:bg-blue-50'

// 危险操作（红色）
compactDangerButtonClass = 'text-red-600 hover:bg-red-50 disabled:cursor-not-allowed disabled:text-slate-400'
```

**使用示例**
```tsx
<div className="flex items-center gap-1">
  <button className={compactSecondaryButtonClass}>编辑</button>
  <button className={compactDangerButtonClass}>删除</button>
</div>
```

**设计原则**
- 操作列按钮优先使用文字，不使用图标，减少视觉占用
- 小屏幕下通过减少内边距和字号来降低行高
- 按钮容器不允许换行，确保操作按钮始终在同一行
- 禁用状态通过降低透明度和禁用鼠标样式来表示

## 七、表格列宽与对齐规范

> 状态：待验证

| 字段 | 建议宽度 | 对齐方式 | 说明 |

## 十、删除确认弹窗规范

> 状态：当前适用

| 属性 | 规范 |
|------|------|
| 宽度 | 400px |
| 高度 | 自适应内容 |
| 定位 | 屏幕居中 |
| 标题 | `确认删除{对象}？`，例如 `确认删除柜体？`、`确认删除摄像头？` |

**展示要求**
- 标题区仅保留风险标识与标题，不在表头区域堆叠长说明，避免弹窗头部过高。
- 风险信息和受影响数据量需在正文区域重点突出。
- 删除说明统一放入正文灰底摘要区，统一使用共享 `ConfirmDialog` 的 `descriptionPlacement="surface"` 形态。
- 弹窗内容优先完整显示；内容增长时可随内容自适应，不要求固定高度。
- 所有列表页的删除操作统一使用同一个二次确认弹窗组件，禁止混用浏览器原生 `window.confirm`。
- 正文文案统一为：`删除后将移除{对象}{关键标识}相关记录，此操作不可恢复。`
- 不再额外堆叠单独的“待删除对象”摘要块；对象名称、编号等关键信息直接写入正文说明句即可。
- 底部按钮顺序统一为：`取消` → `确认删除`
- `确认删除` 按钮统一使用危险样式，文案保持一致；处理中显示 `处理中...`
- 柜体、摄像头、项目内子资源等所有破坏性操作必须遵循同一交互和文案结构，避免页面间确认方式不一致。

## 十八、验收基线

> 状态：当前适用

### 18.1 布局验收

- 页面在桌面端打开后，主体结构需一屏展示完成，不出现页面级纵向滚动
- 左侧菜单支持收起/展开，过渡动画流畅，无内容闪烁
- 列表页顶部操作区、表头、表体可视区域、分页区同时可见
- 列表区在常规桌面分辨率下至少展示 5 行数据
- 新建/编辑弹窗打开后，标题栏与底部按钮区始终可见
- 详情页双栏布局稳定，右侧内容区可随剩余空间伸缩

### 18.2 交互验收

- 表格行悬停时有明显视觉反馈
- 按钮点击有 0.2s 过渡效果
- 输入框聚焦时边框颜色变化
- 状态标签颜色符合规范
- 操作按钮悬停显示下划线

### 18.3 响应式验收

- 左侧菜单收起后，主内容区宽度自动调整
- 表格列宽根据容器宽度自适应
- 最小分辨率 1366×768 下，页面布局正常
- 低于最小分辨率时，出现滚动条但不影响核心功能
- **表格内容在不同屏幕尺寸下保持单行显示，超出部分显示省略号**
- **表格单元格间距根据屏幕尺寸自适应调整（小屏更紧凑）**
- **操作列按钮在小屏幕下使用紧凑样式，减少行高，提高数据可见性**
- **按钮内边距和字号根据屏幕尺寸响应式调整**
- **分页控件支持切换每页显示条数（8/10/20/30/50/100），切换后自动跳转第 1 页**

### 18.4 视觉验收

- 颜色使用符合规范，无自定义颜色
- 字体大小、字重符合规范
- 间距、留白符合规范
