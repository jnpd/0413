# 权限控制契约

## 1. 目的

统一页面权限、按钮权限、接口权限的表达方式，避免权限边界漂移。

## 2. 固定层次

- 页面权限：是否可进入页面
- 按钮权限：是否可看到或点击某个操作
- 接口权限：是否允许真正执行请求

## 3. 固定约束

- 权限最终以后端为准
- 前端权限控制只做体验优化，不做最终安全保证
- 无权限时必须有统一反馈
- 涉及租户、组织、角色边界时，后端必须再次校验

## 4. 页面行为约定

- 无页面权限：进入无权限页或统一提示
- 无按钮权限：默认隐藏或禁用，项目内保持一致
- 无接口权限：后端返回统一权限错误码，前端统一处理

## 5. 统一错误语义

- `40301`: 无页面权限
- `40302`: 无操作权限
- `40303`: 无数据权限

项目可以覆盖具体码值，但必须保持语义稳定。

## 6. AI 使用要求

- 只要页面出现“看”和“操作”的差异，就必须拆成页面权限和按钮权限
- 只要涉及跨租户、跨组织数据，默认后端必须做数据范围校验
